# DEC-019: PR #25 회고 deferred findings 처분 — Q-037~Q-041 일괄 resolution

## Context

PR #25 (adversarial review iterations) + PR #26 (codex review) 에서 식별된
19 개 deferred findings 중 5 개를 별도 슬라이스로 promote 하면서 Q-037 ~
Q-041 entry 를 생성했다 (PR #28 docs(retro) 머지됨, 2026-05-13).

각 Q 의 본문 "사용자 결정 필요 항목" 에 후보 옵션과 기본 권고를 적었다.
사용자가 "전부 네 권고대로" 로 일괄 채택 → 본 DEC 가 5 개 Q 의
resolution 을 단일 lightweight decision 으로 묶는다.

## Decision

### Q-037 — AccessIntervention resolveIntervention importance_score race

채택: **옵션 (2) + (a)**.

- (2) v0 single-operator 가정을 implicit 으로 유지하되, multi-reviewer /
  web UI 도입 전 hardening 필수. INFRA-1B.6.x 슬라이스로 promote (이미
  IMPLEMENTATION_PLAN 등록됨).
- (a) hardening 시 `apoc.lock.nodes([i])` 채택 — Q-024 에서 v0 APOC
  standard adoption 결정됨 → 자연 호환. CAS pattern (b) 와 외부 lock (c)
  는 reject (Cypher round-trip 추가 / 새 storage layer 비용 정당화 어려움).

### Q-038 — discovery_queue multi-worker worker_id CAS

채택: **옵션 (2)**. INFRA-1B.3.x 슬라이스로 promote (이미 IMPLEMENTATION_PLAN
등록됨).

- 사유: v0 single-worker 환경에서도 stale-reclaim 메커니즘 자체가 race
  window 의 존재를 implicit 하게 가정하므로, "single-worker only" invariant
  로 lock 하는 것은 사실 부정확. 다음 INFRA slice 에서 `discovery_queue.worker_id`
  컬럼 추가 + 모든 UPDATE WHERE 에 `worker_id = $self` CAS 조건 적용.

### Q-039 — pollEligibleSources Phase 1/2 streaming

채택: **(a) chunked allSettled** + INFRA-1B.2.x 슬라이스로 promote.

- 사유: v0 scale (72 source × 5MB max = 360MB worst-case) 에서는 영향
  없지만 source pool 확장 시 GC 부담 증가. 옵션 (a) 가 v0 코드와 가장
  호환되고 implement 비용 최소. async iterator (b) / p-queue work-stealing
  (c) 는 reject.

### Q-040 — v0→production schema migration framework

채택: **(a) + (b) + (c) 전부**.

- **(a) v0 운영 contract 명문화 위치**: `docs/05_RUNBOOK.md` 에
  "Pre-deploy schema migration contract" 섹션 추가 (본 PR 에서 처리).
  새 ADR 까지는 불필요 — 운영 contract 수준이지 architectural lock 까지는
  아니다.
- **(b) production-ready entry milestone**: 별도 `P1-MVP-prep` milestone
  + `DEPLOY-1A` 트랙 (이미 IMPLEMENTATION_PLAN 등록됨). PUB-1A.5 turn-key
  MVP gate 와 분리 — turn-key MVP 자체는 v0 deploy 가 아닌 sandbox-gate
  이고, production deploy 는 post-M6 hardening slice 들의 안정화 이후에
  진입.
- **(c) backfill scripts 위치**: `migrations/sqlite/backfill/<NN>_<purpose>.sql`
  네이밍 컨벤션 lock. 일반 forward migration (v7, v8, ...) 과 분리하여
  data-만-수정 vs schema-만-수정 boundary 를 명확히 유지.

### Q-041 — updated_at 형식 CHECK constraints

채택: **millis-bearing 일관 통일** + DEPLOY-1A.2 슬라이스에서 일괄 적용
(이미 IMPLEMENTATION_PLAN 등록됨).

- 사유: `Date#toISOString()` 이 JS 표준 + 대다수 writer 가 이미 millis
  포함 ISO 사용 (crawl_state.last_polled_at, crawl_state.next_eligible_at,
  raw_cache_items.expires_at). 적은 수의 writer (discovery_queue.*) 를
  millis-bearing 으로 맞추는 것이 비용 최소.
- DEPLOY-1A.2 진입 순서:
  1. 모든 writer (snapshot-fingerprint `SET_UPDATED_AT` 상수, v6 schema
     DEFAULT, 등) 를 `strftime('%Y-%m-%dT%H:%M:%fZ', ...)` 또는
     `new Date().toISOString()` 으로 통일하는 PR 선행
  2. CHECK constraint `GLOB '????-??-??T??:??:??.???Z'` 를 모든 ISO
     timestamp 컬럼에 적용 (CREATE TABLE inline for greenfield, 12-step
     rebuild for production)

## Consequences

### 즉시 효과 (본 PR)

- 5 개 Q 모두 `status: resolved`, `resolution: DEC-019` 로 갱신.
- `docs/05_RUNBOOK.md` 에 "Pre-deploy schema migration contract" 섹션 추가
  — v0 단계에서 "wipe-and-reseed acceptable" 운영 contract 명문화.
- IMPLEMENTATION_PLAN.md 의 5 개 슬라이스 (`INFRA-1B.6.x` / `INFRA-1B.3.x`
  / `INFRA-1B.2.x` / `DEPLOY-1A.1` / `DEPLOY-1A.2`) 는 본 DEC 의
  resolution 을 evidence 로 가지고 `planned` status 유지.

### 후속 슬라이스 obligation

각 슬라이스 진입 시 본 DEC 의 채택 옵션을 implementation guidance 로
사용한다. 예: INFRA-1B.6.x 진입 시 apoc.lock.nodes pattern 으로 바로
implement (Q-037 옵션 (a) 가 lock 됨).

### Reject 된 옵션

- Q-037 옵션 (b) CAS pattern: round-trip 추가 비용 정당화 어려움
- Q-037 옵션 (c) 외부 advisory lock: 새 storage layer 비용
- Q-038 옵션 "single-worker only invariant lock": stale-reclaim 존재
  자체가 multi-worker 가정을 implicit 하게 깔고 있어 사실 부정확
- Q-038 옵션 (b) heartbeat 단축: race window 좁아지지만 닫히지 않음
- Q-038 옵션 (c) Redis distributed lock: 새 dependency 과도
- Q-039 옵션 (b) async iterator / (c) p-queue: 옵션 (a) 대비 implement
  비용 더 큼
- Q-040 "단일 ADR 로 통합": 운영 contract 수준이지 architectural lock 까지는
  불필요
- Q-041 옵션 "no-millis 통일": 현재 writer 분포상 millis-bearing 통일이
  변경 최소
