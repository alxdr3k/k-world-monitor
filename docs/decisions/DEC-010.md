# DEC-010: LLM routing v2 lock

## Context

Q-028 의 1차 resolution (DEC-008) 은 Anthropic-only 3-tier (Haiku 1차 +
Sonnet escalate + Opus 선택적) 라우팅 + KPI 5개로 lock 됐다. 2026-05-11
사용자 결정으로 4 가지 변경 필요:

1. system LLM router default vendor = OpenAI (GPT)
2. Tier 0 동 tier = GPT-5.5 Pro xthink + Opus 4.7 xhigh effort (성능/특성
   기준 매핑, 가격 단독 매핑 금지)
3. Google (Gemini) 는 메인/리뷰 X, 탐색 + Tier 3 비용효율 우위 시만
4. Cross-vendor review 강제 단계는 3 종 최소화 (전체 강제 X)

또한 데이터셋 처리에 deterministic 데이터 사이언스 모듈 (ADR-0024) 도입.

본 DEC 는 ADR-0023 + ADR-0024 의 운영 정책을 lock 하고, DEC-008 / ADR-0006
을 supersede.

## Decision

### Core lock items (9)

1. **모델 라우팅 4-tier × multi-vendor** (성능/분야/특성 매핑, ADR-0023
   §1):

   | Tier | OpenAI (default) | Anthropic (override + cross-review) | Google (탐색 + Tier 3 비용효율) |
   |---|---|---|---|
   | Tier 0 | GPT-5.5 Pro extended thinking | Opus 4.7 xhigh effort (cross-review only) | 사용 금지 |
   | Tier 1 | GPT-5.5 Pro standard | Sonnet 4.6 high (한국어 long-context ≥100K 시 override) | 사용 금지 |
   | Tier 2 | GPT-5 mini | Sonnet 4.6 standard (한국어 + 1만 tokens 이상 시 override) | 사용 금지 |
   | Tier 3 | GPT-5 nano | Haiku 4.5 (cross-review only) | Gemini 2.5 Flash (Search grounding + 비용효율 우위 시만) |

2. **Default vendor = OpenAI** (모든 단계). Anthropic override 조건:
   (a) 운영 데이터 200+ 건 SPIKE 로 quality 5%+ 우위 입증, (b) cross-vendor
   review 의무 단계, (c) 사용자 명시 high-stakes flag.

3. **Cross-vendor review 강제 단계 3 종** (그 외 의무 X):
   - Publication preflight cite check overclaim LLM judge: GPT-5 nano →
     Haiku 4.5 review
   - Scenario validate (3') adversarial pass: GPT-5.5 Pro xthink → Opus
     4.7 xhigh review
   - High-stakes Thesis (운영자 명시 flag): GPT-5.5 Pro xthink → Opus 4.7
     xhigh review

4. **Google (Gemini) 사용 scope**:
   - Discovery layer 의 Google Search grounding 보조 (citation 포함 검색
     결과). source registry 의 새 publisher 발견 시 fallback.
   - Tier 3 structured 작업에서 GPT-5 nano 대비 비용효율 우위 입증 시 fallback
     (운영 데이터 누적 + SPIKE 후).
   - 메인 generator / cross-vendor reviewer / Tier 0~2 default 사용 절대
     금지.

5. **Data Science Module 통과 의무** (ADR-0024):
   - 1000+ rows 또는 50KB+ raw payload dataset → Polars / DuckDB /
     statsmodels / scipy stack 통과 → derived metric 만 LLM 입력
   - `derived_metric_ledger` 테이블 (SQLite) 에 reproducibility 3-tuple
     (`dataset_vintage_id` + `spec_sha256` + `library_version_lock_sha256`)
     기록
   - LLM 호출에 raw dataset 발견 시 assertion fail

6. **Prompt caching layering** (ADR-0006 INV-0006-3 그대로, vendor 추상화):
   - 5 단계: tools → system → glossary → source_policy → few-shot →
     [breakpoint] → dynamic
   - TTL: batch 5분, sync 1시간 (vendor 별 native 지원)

7. **Batch API ON**: extract (article/report) / dossier 합성 / EvidencePack
   / ContentDraft (v0 blog_long only — DEC-005 boundary). Sync: Scenario
   composer/validate / Thesis composer / cite check preflight gate.

8. **Strict schema + quote substring 검증** (vendor 추상화):
   - OpenAI `response_format: json_schema`, Anthropic `strict: true`,
     Google `responseSchema`
   - post-LLM quote substring 검증 의무 (faithfulness_rate ≥ 0.99 KPI)
   - temperature: 0 (extract / cite check / thesis), 0.3 (scenario branch)

9. **Cost ceiling 재산정** (DEC-008 ceiling supersede):
   - **soft throttle $5/day** — 큐 backoff + 알람 (DEC-008 그대로 유지)
   - **hard stop $7.5/day** — 신규 LLM 호출 차단 (DEC-008 그대로)
   - **weekly $25** (DEC-008 그대로)
   - **Tier 0 호출 일일 cap = 5회** (GPT-5.5 Pro xthink + Opus 4.7 xhigh
     합산. scenario validate + high-stakes thesis 합산 일평균 2~3회 예상.
     5회 초과 시 알람 + 큐 backoff)
   - **backfill 별도 budget bucket** (DEC-008 그대로, `backfill_run_id` 태그)
   - 1~2주 운영 후 실측 → 재조정 (별도 DEC 또는 본 DEC 갱신)

### KPI 6개 (DEC-008 KPI 5 + cross_vendor_review_coverage)

| KPI | 정의 | 목표 |
|---|---|---|
| cost_per_promoted_claim | (일별 LLM cost) ÷ (promoted claim 수) | ≤ $0.005/claim |
| faithfulness_rate | quote_reason 이 원문 substring 일치 비율 | ≥ 0.99 |
| auto_confirm_fp_rate | reliability_tier=high + confidence ≥ 0.85 auto-confirm 중 reviewer 사후 부정확 비율 | ≤ 0.05 |
| cache_hit_rate | vendor 별 cached_tokens / tokens_in (일별 합산) | ≥ 0.70 (vendor 별) |
| counterclaim_coverage | scenario branch 당 polarity 반대 evidence ≥ 1 비율 | ≥ 0.90 |
| **cross_vendor_review_coverage** (신규) | 강제 3 종 단계의 cross-vendor review 실행률 | ≥ 0.95 |

**soft $5/day ceiling 안에서 충분히 운영 가능**. 단 Tier 0 일일 cap (5회)

### Specialty 도메인 list (DEC-008 그대로)

## Rationale

- **Frontier reasoning 우위 확보**: GPT-5.5 Pro xthink 가 일반 frontier
  reasoning SOTA, Opus 4.7 xhigh 가 source-grounded + agentic long-task
  SOTA. 두 모델 동 tier dual-vendor 가 quality 최대화.
- **Cost 보호**: cross-vendor review 를 3 종 (preflight + scenario validate
  + high-stakes thesis) 으로 한정 — 전체 강제 시 cost 2x 이상 폭발. 사용자
  결정으로 minimum scope.
- **Data Science Module 의 cost / quality 이중 보호**: 토큰 폭발 차단 +
  numerical reasoning 신뢰도 ↑ + reproducibility 확보. dataset 처리에서
  LLM 사용 ↓ → cross-vendor 추가 비용 일부 상쇄.
- **Google 의 Search grounding 만 활용**: vendor 분산 보험 (3-vendor 인프라
  존재) + 메인 신뢰도는 GPT/Anthropic dual-vendor 로 유지. Gemini 의 메인
  사용 신뢰도 검증 불충분 (사용자 결정).
- **Default = OpenAI**: 단일 default vendor 로 운영 단순. 도메인 override
  결정은 운영 데이터 누적 후 SPIKE 로.
