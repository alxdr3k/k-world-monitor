# DEC-003: INFRA-1A.1 재작성 — Round 25 canonical (Neo4j + 9-stage + bidirectional framing) 반영

## Context

이 repo는 2026-05-11 초기 부트스트랩 시 ideation `research-content-pipeline-
architecture`의 **Round 3 lock**(7-stage / SQLite canonical / R2 raw archive /
Markdown promoted only)만 반영해 ADR-0003~0010과 PRD/HLD를 작성했다.
`docs/context/current-state.md:21`에 "Round 3 lock 결정을 구현하는 외부 코드
repo"라고 명시되어 있다.

그러나 ideation은 Round 4~25에서 시스템 정체성·스택·객체 모델·정책을 대대적으로
supersede했다:

- Round 4: 외부 repo 형 결정 (DEC-001로 기록)
- Round 6/Q1: 9-stage 객체 모델 (Source + Thesis 추가) — ADR-0011
- Round 8/Q3: Snapshot 의미 변경 (binary → fingerprint), 시스템 정체성 "출처
  추적 콘텐츠 생산" — ADR-0012
- Round 10/Q5: quote nullable + quote_reason + storage_level 4단계 — ADR-0015
- Round 14/Q9-4: raw cloud upload 영구 금지 (raw_cloud_policy=always_prohibited)
  — ADR-0012
- Round 14/Q9-5: source policy gate + manual_claim_entry — ADR-0017, ADR-0018
- Round 15/Q10-3: metrics framework 6 카테고리 — ADR-0020
- Round 15/Q10-4: policy learning rule-based — ADR-0021
- Round 17/Q12-1: Tier A-D + collectability_score — ADR-0016
- Round 18: Q18 사용자 결정 — **Neo4j Community Edition 채택** — ADR-0012,
  ADR-0013
- Round 18: mode-aware policy gate + access_interventions — ADR-0017
- Round 19: ADR-0015(vendor-neutral) 폐기 + Neo4j-native feature 활용 —
  ADR-0014
- Round 20: Q5 finalize → repo 이름 `alxdr3k/k-world-monitor` (DEC-001 보강)
- Round 21: TL;DR / Brief / Rubric non-archival 갱신 + Current Canonical
  Direction 섹션
- Round 22: 카테고리 8개(core 7 + `digital_assets`) — Q-022
- Round 23/Round 25: bidirectional framing — scenario.impact_targets +
  thesis.stance + market_stance + EvidencePack multi-section + 4축 병렬 시스템
  정체성 — ADR-0019
- Round 24: 자체 audit + R23 정정 + Q-024~Q-028 신규

특히 ADR-0007 Alternative D는 "그래프 DB 별도 도입(e.g., Neo4j) — discarded ...
외부 의존성 최소화 위반"이라고 명시 거부했지만, R18 사용자 결정으로 Neo4j 채택
이 lock됐다.

사용자는 2026-05-11 drift 검토 후 "Reset 옵션" 선택 — INFRA-1A.1을 Round 25
canonical로 재작성.

## Decision

다음을 수행한다:

1. **새 ADR 11개 작성** (Round 4~25 결정 반영):
   - ADR-0011 9-stage 객체 모델 (Source + Thesis) — supersedes ADR-0003
   - ADR-0012 Non-archival storage (Neo4j + SQLite + R2 permitted artifacts +
     raw cloud prohibition) — supersedes ADR-0004
   - ADR-0013 Edge ledger via Neo4j typed relationships — supersedes ADR-0007
   - ADR-0014 Neo4j-native feature adoption (APOC + GDS + native vector +
     native FTS) + intentional lock-in
   - ADR-0015 Evidence (nullable + quote_reason + storage_level) + cite check
     5+1 — supersedes ADR-0008
   - ADR-0016 Collection realism (Tier A-D + collectability_score + no bot
     bypass)
   - ADR-0017 Source policy gate (mode-aware) + access_interventions
   - ADR-0018 Manual feedback inbound workflow (3-way 분리 + CLI 5종)
   - ADR-0019 Bidirectional framing (scenario impact_targets, thesis stance +
     market_stance, EvidencePack multi-section)
   - ADR-0020 System metrics framework (6 카테고리 + evaluation harness, 단계화)
   - ADR-0021 Policy learning framework (rule-based, auto-tighten allowed /
     auto-relax prohibited)

2. **ADR-0003 / ADR-0004 / ADR-0007 / ADR-0008을 superseded 상태로 표시** +
   본문에 supersede 사유 명시.

3. **PRD 갱신**: 9-stage REQ-001, Neo4j REQ-002, raw cloud prohibition REQ-003,
   evidence nullable REQ-007, Neo4j typed edges REQ-008, REQ-017~REQ-026 신규
   (Tier A-D / policy gate / access_intervention / manual feedback / scenario
   impact_targets / thesis stance / EvidencePack / metrics / policy learning /
   카테고리 8개), NFR-008/NFR-009 신규 (legal safety / bidirectional balance).

4. **HLD 갱신**: Neo4j Component 추가, Thesis Component 추가, Policy Gate
   mode-aware, Access Intervention Queue, Manual Feedback CLI, Metrics
   Collector, Policy Learner. Storage seams 4 layer (Neo4j / SQLite /
   R2 permitted / Markdown).

5. **current-state.md 갱신**: "Round 3 lock 결정" → "Current Canonical Direction
   (Round 1~25 누적)" 참조. 미해결 Q-020~Q-030 추가. SPIKE-001 대상 갱신
   (SQLite+FTS5 → Neo4j).

6. **04_IMPLEMENTATION_PLAN.md 갱신**: INFRA-1A.1 재작성 후 landed 표시 + 신규
   슬라이스 (INFRA-1A.5/.6/.7/.8, INFRA-1B.5/.6/.7, AGG-1A.4, OPS-1A.3/.4) 추가.
   Track AGG에 thesis 추가, OPS에 metrics + policy learning 추가.

7. **글로서리 신규**: source, thesis, access_intervention, manual_claim_entry,
   policy_gate, collectability_score, raw_cloud_policy (7개). impact_target /
   transmission_channel / source_perspective는 v1 schema migration 시점에
   추가.

## Rationale

- Round 3 lock만 반영된 ADR/PRD/HLD는 R18 Neo4j 결정과 정면 충돌 (ADR-0007
  Alternative D가 Neo4j를 명시 거부). 사용자 결정을 반영하려면 supersede 필수.
- Round 25 canonical은 ideation 본문의 "Current Canonical Direction" 섹션에서
  명시적으로 라운드 본문보다 우선한다고 선언되어 있다. drift 정합성 회복이
  필요.
- INFRA-1A.1을 supersede 표시만 하고 별도 새 슬라이스로 재진입하는 것보다
  같은 슬라이스를 Round 25 canonical 기준으로 재작성하는 것이 roadmap에
  덜 침입적 — 후속 슬라이스의 의존 관계가 깨지지 않음.
- Q-020~Q-030(신규 11개)은 v0 부트스트랩 진입을 막지 않음 — 구현 단계에서
  발견되는 trade-off는 별도 ADR/DEC.
