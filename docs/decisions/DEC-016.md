# DEC-016: Neo4j 연결 풀 설정 환경변수화

## Context

`src/storage/neo4j/connection.ts:14` 에 `maxConnectionPoolSize: 10`,
`connectionAcquisitionTimeout: 5000` 가 하드코딩되어 있다.
Opus 리뷰 Q1: Discovery worker 에서 10+ 동시 `withSession()` 호출 시 타임아웃.
현재 설정 변경은 코드 수정 + 재배포가 필요하다.

## Decision

```typescript
// src/storage/neo4j/connection.ts
const driver = neo4j.driver(uri, auth, {
  maxConnectionPoolSize: parseInt(process.env["NEO4J_MAX_POOL_SIZE"] ?? "10"),
  connectionAcquisitionTimeout: parseInt(process.env["NEO4J_ACQ_TIMEOUT_MS"] ?? "5000"),
});
```

기본값은 현재 하드코딩 값과 동일 — 동작 변화 없음.

## Rationale

- 환경변수화로 재배포 없이 운영 조정 가능
- Discovery worker 병렬도 증가 시 `NEO4J_MAX_POOL_SIZE=20` 으로 조정 가능
- 기본값 유지로 기존 테스트/운영 영향 없음

## Consequences

- 긍정: 운영 유연성 증가, 재배포 없는 pool 조정
- 부정: 없음
- Follow-ups: `docs/current/OPERATIONS.md` 에 env var 추가 (INFRA-1B.2 진입 전)
