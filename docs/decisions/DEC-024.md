# DEC-024: 2026-05-15 adversarial review action decisions

## Context

PR #40 (2026-05-15) 가 Claude Opus 4.7 의 12-layer adversarial review (208 finding) + GPT cross-review (~80 finding) 의 합리적 종합을 retrospective artifact 로 등록하면서 **Q-052~Q-059 를 open placeholder Q** 로 신설. 8 Q 모두 운영자 결정 pending 상태로 두면 다음 agent 가 다시 판단해야 하고 "deferred" 자체가 결정으로 명시되지 않으면 doc drift risk.

본 DEC 는 Q-052~Q-059 를 **resolved (6) + deferred (2) 명시 결정** 으로 일괄 처리한다. defer 자체도 명시 결정 (re-entry gate + 조건 lock) 으로 기록.

## Decision

### D1 — Q-052 main branch protection 3중 정책 충돌

**Decision**: DEC-020 우선. **PR-only main branch policy** 가 canonical target.

```text
Policy target: PR-only + branch protection.
Actual GH state: admin task pending until branch protection is registered.
```

세부:
- `docs/01_PRD.md:208-212` + DEC-020 Q-048 의 "main branch protection PR-only" canonical 유지
- `CLAUDE.md` global "Git Branch Policy" 의 k-world-monitor 의 main 직접 push 예외 — **운영자 admin task 로 제거 또는 break-glass 만 허용 으로 축소** (CLAUDE.md 는 user-global 파일이라 본 repo 의 commit scope 외 — operator 가 ~/.claude/CLAUDE.md 직접 편집)
- 실제 GitHub branch protection 등록 — **운영자 admin task** (`gh api -X PUT /repos/.../branches/main/protection ...`)
- 미등록 동안 docs 는 "policy target: PR-only; admin task pending" 으로 표기 (이미 enforce 된 것 처럼 적지 말 것)

**Break-glass 절차** (direct push 예외 허용 사례):
- security credential leak 즉시 제거
- broken main 이 모든 PR check 차단 시 복구
- operator explicit emergency action

Break-glass 후 의무:
- retrospective note 기록 (`docs/10_PROJECT_RETROSPECTIVE.md`)
- follow-up PR 또는 repair commit

**Resolves**: Q-052.

### D2 — Q-053 chunker raw text persistence 정책

**Decision**: **Option (a) — `archive_policy` 기반 gate**. v0 에서 신규 `local_storage_policy` enum 도입 안 함.

`chunkSnapshot(input)` 의무 input:
- `sourceId`
- `archivePolicy`

Gate logic:
| archive_policy | 결과 |
|---|---|
| `do_not_collect` | reject |
| `metadata_only` | reject |
| `excerpt_only` | reject full-text chunking (excerpt-limited chunking 은 별도 future design 전까지 금지) |
| `full_snapshot_allowed` | allow |

**Resolves**: Q-053.

### D3 — Q-054 Source Registry canonical store

**Decision**: **Option (b) — SQLite `source_profile` table** 을 canonical metadata store 로 신설. **Neo4j Source node 는 graph traversal projection**.

`source_profile` schema (D7 의 `source_role` 흡수):

```sql
CREATE TABLE source_profile (
  source_id          TEXT PRIMARY KEY
    REFERENCES source_material_policy(source_id),
  slug               TEXT NOT NULL UNIQUE,
  name               TEXT NOT NULL,
  publisher          TEXT NOT NULL,
  url                TEXT NOT NULL,
  rss_url            TEXT,
  api_base           TEXT,
  access_method      TEXT NOT NULL,
  reliability_tier   INTEGER NOT NULL,
  source_perspective TEXT NOT NULL,
  source_role        TEXT NOT NULL,       -- D7 흡수
  meta_category      TEXT NOT NULL,
  subtopic_tags_json TEXT NOT NULL,
  collectability_json TEXT NOT NULL,
  notes              TEXT,
  active_v0          INTEGER NOT NULL DEFAULT 1,
  updated_at         TEXT NOT NULL
);
```

이유:
- `source_material_policy` 와 같은 SQLite transaction 안 upsert 가능
- source quality dashboard 가 SQLite query 로 단순
- seed idempotency 단순
- Neo4j 는 graph traversal projection 유지 (cross-store mutation seam 감소)

**Resolves**: Q-054.

### D4 — Q-055 dataset ingestion MVP timing

**Decision**: **Option (a) — ADR-0024 / EXTR-1A.5 timing 유지**. M3 이전 dataset MVP **도입 안 함**.

Re-entry gate: **PUB-1A.5 첫 발행 retrospective** 시점에 RSS-only output 품질 실측 후 재평가. ADR-0024 의 Polars + DuckDB + statsmodels + scipy stack lock 변경 없음.

**Defers**: Q-055 (re-entry gate 명시 = PUB-1A.5 retrospective + RSS-only output quality evidence).

### D6 — Q-057 Claim promotion / semantic dedup / claim_direction

**Decision**: **Option (b) — 신규 ADR (가칭 ADR-NNNN (다음 가용 ID) "Claim promotion, semantic dedup, claim direction") 로 lock**. **EXTR-1A.1 진입 직전 accepted 상태 의무**.

본 PR (#40) 에서는 ADR 작성 안 함. Q-057 은 **deferred** 상태 + blocker 명시.

ADR-NNNN (다음 가용 ID) (가칭) 필수 invariant:
- Claim 은 `raw_claim_statement` 보존
- `normalized_claim_key` 별도 필드
- `claim_direction` enum (Thesis stance 와 separate axis: risk_increasing / risk_decreasing / opportunity_increasing / opportunity_decreasing / neutral_fact / ambiguous)
- `confidence_score` vs `importance_score` 분리
- Dossier/Scenario promotion 은 `promotion_score` threshold 통과 의무
- semantic duplicate claim 은 별도 Claim node 안 만들고 **evidence anchor 다수 attach** pattern

**Defers**: Q-057 (re-entry gate = ADR-NNNN (다음 가용 ID) (가칭) 작성 + accepted, EXTR-1A.1 진입 직전).

### D7 — Q-058 source_role multi-dim

**Decision**: **Option (a) — Q-054 source_profile slice 안에 흡수**. 별도 ADR 불필요.

v0 `source_role` enum (single primary, required):

- `official_policy`
- `wire_news`
- `expert_analysis`
- `market_commentary`
- `local_observer`
- `opposition_view`

- `academic`

Multi-role tags 는 v1+ deferred (현 v0 = single primary 만).

**Resolves**: Q-058.

### D8 — Q-059 attention budget — P2/P3 backlog 처리 정책

**Decision**: **Option (d) — hybrid prioritization**.

Rule:
1. **P0 / P1 blockers 만** P0-M2 gate accept 전에 처리 가능
2. **각 P1 slice 가 인접 P2 finding 흡수 가능** (efficient batching)
3. **Standalone P2 / P3 backlog 는 P0-M2 gate 전에 frozen** — operator explicit promotion 시만 진입
4. **잔여 P3 batch 는 PUB-1A.5 첫 발행 retrospective 까지 archive**
5. **AI-P0-1 (`INFRA-1B.3.h1-policy-fix`) 가 최우선 engineering task**

**Resolves**: Q-059.
