# DEC-005: v0 turn-key publish scope — blog_long only, TTS deferred, manual cross-post, manual correction approve

## Context

ADR-0011은 ContentDraft 4-format(blog_long / youtube_long / shorts /
newsletter)을 Thesis 재사용 anchor로 lock했다. ADR-0022는 자체 사이트
(Astro 5.0 + Cloudflare Pages) 를 publishing primary로 lock했다. 그러나
v0 turn-key 발행 milestone (P0-M6 thin slice) 은 1인 + LLM agent 시간 박싱
환경에서 4-format을 모두 generate / 외부 플랫폼 자동 cross-post / TTS /
자동 correction 까지 한 번에 켤 수 없다.

본 결정은 v0 turn-key publish의 **활성 표면**을 단일 format + manual workflow
로 제한해 첫 publication까지의 시간 박싱을 가능하게 한다. 4-format reuse
anchor와 cite check 5+1 정책은 schema-level lock 유지.

## Decision

v0 turn-key publish scope를 다음으로 lock한다:

1. **활성 generation format**: `blog_long` 1개만. youtube_long / shorts /
   newsletter는 v1+ (Q-032). 4-format anchor는 ContentDraft schema lock
   유지 (ADR-0011 INV 보존), v0에서 generation만 blog_long.
2. **발행 채널**: 자체 사이트 (Astro 5.0 + Cloudflare Pages, ADR-0022)
   단일.
3. **외부 플랫폼 cross-post**: manual. Substack / YouTube / X에 운영자가
   직접 게시하되, 외부 게시물의 cite footnote는 모두 자체 사이트 URL을
   canonical anchor로 가리킴 (ADR-0022 INV-0022-2). 자동 cross-post + API
   integration은 v1+ (Q-033).
4. **TTS**: v0 scope out. youtube_long / shorts 의 script → audio 변환은
   외주 또는 v1+ (Q-031에서 결정). v0 turn-key 단계에서는 음성 콘텐츠
   발행 자체가 deferred.
5. **Correction / retraction workflow**: manual approve. ADR-0018 manual
   feedback CLI (`pipeline feedback` / `pipeline intervention review`) 를
   correction approve workflow의 기반으로 확장. ContentDraft frontmatter
   `status` 전이 (live → corrected → retracted) 와 `correction_ledger[]`
   append는 운영자 명시 승인. auto retraction trigger는 v1+ (Q-034).
6. **Evaluation harness**: ADR-0020 staged v0/v1/v2 에 따라 v0는 mock-first
   gold query set. real LLM 평가 자동화는 v1+. cite check 5+1 자체는
   build-time enforce (ADR-0022 INV-0022-3).

v0 turn-key 시간 박싱: **2주 lock (목표)**. 1인 + LLM agent 환경에서
10-stage canonical (ADR-0025 supersedes ADR-0011) 부트스트랩 + 자체 사이트 + 첫 발행 1건을 2주 목표로
추진. 진행 중 slice 단위 capacity 추적 — 목표 미달성 시 별도 trigger
로 reflow 검토 (DEC 또는 IMPL plan slice 갱신).

## Rationale

- **4-format 동시 generation은 v0 시간 박싱 불가**: blog_long ≈ 자체 사이트
  HTML 렌더 + cite footnote 컴포넌트, youtube_long ≈ script + chapter +
  TTS + chart 영상, shorts ≈ 60초 script + cut, newsletter ≈ email-safe
  HTML + Substack export. 각각 별도 composer가 필요 — 첫 발행까지 시간
  박싱하려면 blog_long만 활성.
- **자동 cross-post는 외부 API + auth 복잡도**: Substack API + YouTube
  Data API + X API + auth + 정정 cascade까지 v0에 포함하면 1인 운영
  대비 over-engineering. manual cross-post 는 footnote anchor만 자체 사이트
  URL로 유지하면 cite check 정합성 보존.
- **TTS는 외주 또는 v1+**: 자체 TTS는 voice model + audio 파일 자체 호스팅
  + RSS audio enclosure 추가 부담. youtube_long / shorts 자체가 v0 out
  이므로 TTS 도 deferred.
- **manual correction approve = ADR-0018 자연 연장**: ADR-0018 의
  manual_claim_entries 3-way 분리 + CLI 3-option 패턴을 correction approve에
  적용. auto retraction trigger 는 false positive 리스크 (예 cite check
  warning이 retraction을 자동 트리거하면 publication state 안정성 ↓) →
  v1+ 에 정책 정밀화 후 검토 (Q-034).
- **Evaluation harness mock-first**: ADR-0020 이미 staged v0/v1/v2 정의.
  v0에서 real LLM eval 자동화까지 가면 cost 부담 + flaky test, mock-first
  는 cite gate + schema validation 만 build-time enforce.

## Consequences

- 긍정:
  - v0 turn-key 발행 milestone 의 표면이 단일 format + 단일 채널로 압축 →
    1인 + LLM agent 시간 박싱 가능 (현실적 2~6주 범위)
  - 4-format anchor schema-level lock 보존 → v1+ activation 시 schema
    migration 불필요
  - manual correction approve 는 ADR-0018 CLI 와 자연 정합 → 신규 도구 추가
    없음
  - 자체 사이트 cite anchor canonical URL 정합 (ADR-0022 INV-0022-2) →
    외부 플랫폼 manual cross-post 도 cite check 정합 유지
- 부정:
  - **2주 목표 lock 의 risk**: 1인 + LLM agent 환경에서 10-stage canonical (ADR-0025 supersedes ADR-0011)
    부트스트랩 + 자체 사이트 + 첫 발행 1건은 capacity 변동에 민감 — 목표
    미달성 시 slice 단위로 정직 reflow (IMPL plan 갱신)
  - youtube_long / shorts / newsletter 발행 부재 → v0 도달 채널 협소
    (자체 사이트 + manual Substack/YouTube/X cross-post 만)
  - TTS deferred → 음성 콘텐츠 발행 자체 v0 부재
  - manual correction approve → 빠른 정정 응답성 제한 (v1+ Q-034 자동화
    검토)
  - 4-format 중 3개가 deferred → ContentDraft Composer 의 format 분기 로직
    구현 우선순위가 blog_long에 편중, 나머지 format composer 는 v1+
    bootstrap 필요
- Follow-ups:
  - Slice `PUB-1A.1` scope 재정의 — v0 활성 format = blog_long, 나머지 3
    format composer는 v1+ slice 분리
  - Slice `PUB-1A.4` 신규 — Astro skeleton + Content Collection + Zod
    schema mirror + cite/retraction/correction 컴포넌트 + Cloudflare Pages
    배포 (ADR-0022)
  - Slice `PUB-1A.5` 신규 — 첫 publication (blog_long 1건) — P0-M6 v0
    turn-key gate
  - PRD REQ-027 신규 — 자체 사이트 publishing primary + v0 turn-key
    phasing
  - Q-031 (TTS v1 timing + provider), Q-032 (4-format auto-generate
    phasing), Q-033 (외부 플랫폼 auto cross-post timing), Q-034 (Auto
    retraction trigger 정책 v1+) — 본 DEC가 모두 새 Q로 분리
