# DEC-022: Q-051 #6 resolution — Research App `/ops` UI stack lock

## Context

Q-051 round 2 에서 사용자가 CLI → Web (mobile-first) pivot 결정 + GPT 31
review 후 Topology B (CF Pages public anchor + Hetzner private /ops + /api,
same-origin) lock. 그러나 `/ops` 의 구체 frontend stack 은 미결정 —
UI-spec §14 #1 "Stack: 옵션 A~F 중 선택" 으로 round 14/15 까지 pending.

본 DEC 는 GPT 32 review (2026-05-14) 와 사용자 동의로 stack lock.

## Decision (lock)

채택: **A' (Astro shell + React island) — 구체 stack 6 layer**.

| Layer | 선택 | 사유 |
|-------|------|------|
| Shell | **Astro 5.x** | ADR-0022 publication site 와 동일 build target — public anchor `/posts/*` + private `/ops/*` 가 같은 `astro.config.*.ts` 변종 안에서 동작. Topology B 의 `astro.config.public.ts` (CF Pages) + `astro.config.ops.ts` (Hetzner) 분리 build, shared schema 일관. |
| Interactive layer | **React 18 island** | `@astrojs/react` integration. `client:load` / `client:visible` / `client:idle` directive 로 page-level hydration cost 제어. /ops 모바일 페이지 1~2 island 기준. LLM (Claude/Codex) coding 안전성 ↑ — React 패턴/예제/디버깅 사례 압도적. |
| Styling | **Tailwind CSS** | 모바일 우선 utility-first. Astro public 사이트와 동일 design token. `tailwind.config.ts` 단일 source. |
| Component primitives | **shadcn/ui + Radix UI** | Radix headless (Dialog/Sheet/Drawer/Popover/DropdownMenu) — 모바일 drawer/sheet 표준. shadcn/ui CLI 로 `src/shared/ui/` 안에 copy-paste 방식 (npm dep weight 낮음, lock 가능 source). |
| Server state / cache | **TanStack Query v5** | server-state-heavy `/ops` 페이지 (sessions / rounds / turns / publications) 의 stale-while-revalidate + cache invalidation 표준. SSE event 수신 시 `queryClient.invalidateQueries(['session', id])` 패턴. ~14KB gzip, 모바일 acceptable. |
| Realtime transport | **SSE (Server-Sent Events)** | Q-050 GPT T-4 (turn_event durable log) 와 일치 — turn / agent progress streaming. WebSocket 불필요 (UI-spec §14 #6 동의). 모바일 reconnect 시 `turn_event` table 로 catch-up. |

## Rationale (GPT 32 review 기반)

1. **Astro anchor 일관성** — ADR-0022 lock 으로 발행 사이트가 Astro 5.x +
   CF Pages 결정 완료. `/ops` 를 별도 framework (Next.js / Remix) 로 도입
   시 build target / schema / design system 이중화 — 1인 운영 repo 에
   maintenance overhead 과도.

2. **`/ops` = AI research console** — Scenario graph (vega-lite / mermaid) +
   route selector + drawer/sheet + SSE progress + cache invalidation 등
   SPA-grade interaction 필요. Astro SSR 단독 (HTMX 패턴) 으로는 부족.
   React island 가 server-rendered shell 위에 interactive component 만
   hydrate — first paint 빠름 + interaction 풍부 양립.

3. **LLM coding 안전성** — Claude/Codex 가 `/ops` 구현·유지보수 주체.
   React 의 패턴/라이브러리/예제/디버깅 사례가 Solid/Svelte/Vue/HTMX
   대비 압도적으로 많음 → LLM 이 생성한 코드의 실제 동작 신뢰도 ↑.

4. **shadcn/ui + Radix** — Tailwind 와 design token 일관. CLI copy-paste
   방식이라 source code 가 repo 안에 lock 됨 (vendor update 위험 0).
   Radix headless 가 a11y 표준 (focus trap / aria-modal / keyboard nav)
   기본 제공 — operator console 신뢰도.

5. **TanStack Query** — SSE 수신 + manual mutation + optimistic update
   패턴이 명시적. turn_event durable log (Q-050) 와 binding 시
   `useQuery(['session', id])` + `EventSource` listener → `setQueryData`
   인플레이스 업데이트 표준.

6. **Solid/Svelte/Vue reject** — Astro 와 island 통합 자체는 가능하나
   LLM coding 안전성 + 커뮤니티 + Astro 공식 권고 stack 측면에서 React
   가 default. Next.js (옵션 B) 는 Astro anchor 와 별도 build 이중화 →
   reject. HTMX (옵션 E, 기존 §14 #1 권고) 는 SPA-grade interaction
   부족 + 사용자 피드백 ("내가 HTMX 모르는 게 문제가 아니라 네가 HTMX
   충분히 아는가") → reject.

## Critique (GPT 권고 비판적 검토 — 본 DEC 반영)

GPT 권고 안에 명시되지 않았으나 본 DEC 작성 과정에서 추가 검토한 사항:

### Critique 1: React island hydration cost — 모바일 LTE 대응

React 18 + Tailwind + Radix runtime + TanStack Query bundle ≈ 70KB gzip
(estimate). 모바일 LTE first paint 영향 가능.

**Mitigation (DEC 안에 lock)**:
- `client:visible` directive 우선 — page 진입 시 viewport 안 island 만
  hydrate. 초기 LCP 비용 ↓.
- `/ops/sessions` list / `/ops/posts/:slug` 같은 read-heavy 페이지는
  Astro SSR + 0 island 가능 (P0-M6 RESEARCH-1A.0 scope 가 정확히 이 형태).
- Scenario graph (`/ops/scenarios/:sid`) 만 `client:load` 의무 — 진입
  즉시 interaction 필요.

### Critique 2: shadcn/ui copy-paste 위치 — 디자인 시스템 anchor

shadcn/ui 는 npm dep 가 아니라 CLI 로 component 를 프로젝트로 copy-paste.
**위치 lock**: `src/shared/ui/` (UI-spec §13 Topology B 의 shared
primitives 디렉토리와 일치).

- Astro public site (`/posts/*`) 와 React island (`/ops/*`) 가 동일
  `src/shared/ui/` 의 Tailwind primitives import.
- 단 shadcn/ui 의 React-only component (Dialog / Sheet 등) 는 React
  island 안에서만 import — Astro static page 는 plain Tailwind class 만.

### Critique 3: TanStack Query phasing — P0 vs P1+

P0-M6 RESEARCH-1A.0 (read-only mobile minimum) 은 **TanStack Query 도입
의무 없음** — Astro SSR + plain `fetch` 만으로 read-only `/ops/sessions`
/ `/ops/sessions/:id` (header only) / `/ops/posts/:slug` 가능.

TanStack Query 도입 시점 = **RESEARCH-1A.1 슬라이스 (P1+) 부터** —
mutation flow (ask / add round / branch) + SSE binding + cache invalidation
이 등장하는 시점.

shadcn/ui CLI scaffold 도 동일 phasing — P0-M6 안에서는 Tailwind primitives
만, P1+ 부터 Radix component 도입.

### Critique 4: ADR-0031 발급 시점 — 본 PR 안 vs 후속

UI-spec §14 line 433 lock — "본 spec 의 다음 항목 사용자 결정 후 정식
ADR 작성". 본 DEC-022 가 stack 결정 자체는 lock 했으나, **ADR-0031
(Research App /ops Stack) 발급은 RESEARCH-1A.1 슬라이스 시작 직전까지
deferred** — 그 시점에 구체 `package.json` dep version /
`astro.config.ops.ts` shape / `client:*` directive policy 확정 가능.
DEC-022 가 ADR 발급 의무 anchor.

**Status update (DEC-022 anchor resolved)**: 본 DEC-022 의 PR (round 18,
PR #33) 안에서 ADR-0031 placeholder stub 만 landed 후, 후속 PR
`claude/adr-0029-research-app-ops-stack` 에서 body 본격 작성 + status:
accepted promote 완료. ADR-0031 본문이 INV-0031-1~7 lock + §3.1~§3.6
구체 implementation guide (package.json caret 범위 / astro.config.ops.ts
shape / client:* directive policy 페이지별 표 / src/shared/ui layout /
ops/lib/query 구조 / Lighthouse bundle budget acceptance gate) 흡수.
본 Critique 4 의 deferred 의무 = closed (RESEARCH-1A.1 슬라이스 시작
직전 의무를 본 ADR accept 가 만족).

### Critique 5: 본 stack 결정과 PWA / Service Worker 호환성

RESEARCH-1A.4 (PWA + offline draft, P1+) 는 React island bundle 의
service worker pre-cache 표준 패턴 (Workbox + Astro adapter). 본 DEC 와
직접 충돌 없음. 단 IndexedDB 의 pending turn queue 가 TanStack Query 의
optimistic update 와 sync 의무 → RESEARCH-1A.4 슬라이스 spec 에서 확정.

### Critique 6: Voice input (Whisper API, RESEARCH-1A.5) 와의 통합

Voice input UI 가 React island 안 `<Button>` + `MediaRecorder` API.
shadcn/ui `<Button variant="ghost">` + Radix Tooltip 으로 구성 — 본 stack
안에서 자연스러움.

## Consequences

### Positive

- **build target 단순화** — Astro 단일 framework, `astro.config.public.ts`
  + `astro.config.ops.ts` 두 변종만.
- **shared schema 일관** — INFRA-1B.7e 의 `src/shared/schema/` Zod schema
  가 Astro Content Collection (public anchor 본문) + Bun API (Hono route
  validation) + React island (form / TanStack Query types) 3 surface 에
  서 동일 import.
- **LLM coding 신뢰도** — Claude/Codex 가 `/ops` 구현 시 React + Tailwind +
  shadcn/ui 패턴 안에서 작성 → 코드 품질 / 디버깅 / 유지보수 표준화.
- **a11y 기본** — Radix headless 가 keyboard nav + focus trap + aria
  표준 제공.

### Negative / 주의

- **모바일 LCP** — Critique 1 mitigation 의무. 페이지별 `client:*`
  directive 정책 RESEARCH-1A.1 슬라이스 안에서 확정.
- **bundle size 관찰** — Lighthouse mobile budget 설정 의무 (P1+
  RESEARCH-1A.1 acceptance gate 안 추가 검토). 단 P0-M6 RESEARCH-1A.0
  은 Astro SSR + 0 island 으로 budget 영향 없음.
- **dep 도입 시점** — shadcn/ui CLI / Radix / TanStack Query 도입은
  RESEARCH-1A.1 시점부터. P0-M6 RESEARCH-1A.0 은 plain Tailwind 만.
- **ADR-0031 발급 의무 = closed** — Critique 4. 본 DEC-022 PR 안에서
  placeholder stub landed 후, 후속 PR (`claude/adr-0029-research-app-
  ops-stack`) 에서 body 본격 작성 + status: accepted promote 완료
  (INV-0031-1~7 + 구체 implementation guide + bundle budget acceptance
  gate). DEC-022 anchor 의무 = 해소.
