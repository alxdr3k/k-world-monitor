# DEC-004: v0 카테고리 4개 (정책 / 경제 / 사회 / 대중문화) — Q-022 resolve

## Context

Q-022는 v0 카테고리를 "core 7 (macro_finance / geopolitics_security /
health_biosecurity / energy_commodities / trade_supply_chain /
climate_environment / technology_cyber_ai) + secondary 1 (digital_assets) =
8개"로 권고하고 PRD REQ-026 / ADR-0019 카테고리 섹션에 lock되어 있다
(`docs/01_PRD.md:129`).

사용자는 v0 turn-key 발행 milestone을 시간 박싱하기 위해 카테고리를 4개로
축소 결정. 8 카테고리는 1인 운영 + LLM agent 환경에서 첫 publication까지의
**Source registry seed 분포 균형 (Q-021)** + **Dossier topic 운영 부담** +
**4-format ContentDraft 발행 빈도** 모두에 곱셈 부담을 준다. 4 메타 카테고리로
축소하면 8 카테고리는 메타 카테고리 안의 하위 tag로 강등되며 v1+ 시점에
재승격 가능.

## Decision

v0 카테고리를 4개로 lock한다:

1. **정책 (policy)** — geopolitics_security / governance_institutions /
   regulatory_policy 흡수
2. **경제 (economy)** — macro_finance / trade_supply_chain /
   energy_commodities / digital_assets 흡수
3. **사회 (society)** — health_biosecurity / demographics_migration /
   food_water_security / climate_environment 흡수
4. **대중문화 (pop_culture)** — technology_cyber_ai의 consumer surface /
   media / cultural trend / 디지털 콘텐츠 생태 흡수

기존 8 카테고리는 4 카테고리 안의 하위 tag로 보존되며, Dossier / Scenario
frontmatter `topic` 필드는 4 enum, `subtopic_tags[]` 필드는 8+개 enum으로
v0 schema에 lock한다. enum yaml은 `data/categories.yaml` (외부 repo seed 시점
commit).

source_perspective 분포 균형(REQ-022)은 4 카테고리에서도 동일 적용 — 각
카테고리당 Tier A source 최소 1개 + risk_observer ≤ 50% / opportunity_observer
≥ 25% / neutral ≥ 15%.

## Rationale

- v0 turn-key 발행 milestone은 첫 publication 1건이 목표. 카테고리 8개 ×
  Tier A seed 30~50개 × Dossier composition × Scenario validate × Thesis ×
  ContentDraft 4-format은 1인 운영에서 시간 박싱 불가능
- 메타 카테고리 4개 + 하위 tag 보존은 8 카테고리 정보를 잃지 않으면서
  Dossier composition 단위를 압축
- 4 메타 카테고리는 1인 콘텐츠 발행자의 일반적 mental model과 정합 (정책 /
  경제 / 사회 / 대중문화) — 외부 독자 navigation에도 적합
- ADR-0019 bidirectional framing(risk / opportunity / resilience /
  asymmetric 4축)은 카테고리 수와 무관 — 4 카테고리 안에서도 4축 병렬 유지
- v1+ 재승격 경로 보존: Dossier가 특정 하위 tag로 누적되면 v1+에 메타
  카테고리에서 독립 카테고리로 승격 (예 digital_assets 누적 충분 시 v2에
  독립 카테고리 복귀)

## Consequences

- 긍정:
  - v0 source registry seed scope 1/2 축소 (8 카테고리 × 5개 → 4 카테고리 ×
    5개 ≈ 20 source) — INFRA-1A.6 진입 부담 ↓
  - Dossier composition 단위 명확 (4개 vs 8개)
  - 외부 독자 navigation 단순화
  - ADR-0019 bidirectional framing 그대로 보존
- 부정:
  - 8 카테고리 도메인 specificity 손실 — Dossier의 정보 밀도 ↓ (v1+에서 하위
    tag 누적 시 재승격)
  - PRD REQ-026 / ADR-0019 categories enum 갱신 비용 (이 DEC + 후속 update)
  - 카테고리 mapping (8 → 4 흡수) 의 운영 ambiguity — 예 climate_environment를
    "사회"로 흡수하는 것은 macro 영향 측면에서 "경제"와 경계가 있음. 운영자
    판단 + subtopic_tags로 보완
- Follow-ups:
  - Slice `INFRA-1A.6` — Tier A seed를 4 카테고리 분포 균형으로 재구성 (Q-021
    완료 전제)
  - PRD REQ-026 update — "v0 4 메타 카테고리 + 8 하위 tag" 명시
  - ADR-0019 categories section update — 4 메타 카테고리 + 4축 병렬 유지
  - `data/categories.yaml` v0 seed commit (외부 repo 시점)
  - Q-032 phasing (8 카테고리 복귀 / 5번째 카테고리 추가 시점)
