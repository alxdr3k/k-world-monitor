# DEC-001: 외부 코드 repo 형으로 운영 (vault 내 형 결정 진화)

## Context

second-brain ideation `research-content-pipeline-architecture` Round 3에서
사용자는 다음을 결정했다.

- 출판: `05. Projects/세계경제-시나리오-콘텐츠-파이프라인/Publications/`
  (vault 내 일반 PARA project)
- 리서치: `06. Areas/세계경제-지정학-감염병-모니터링/`
- Bulk store: 위 area 산하 `_store/research.db` + `_store/*.jsonl`

그 후 실제 부트스트랩은 외부 코드 repo (`alxdr3k/k-world-monitor`, boilerplate
greenfield)에서 진행되었다. 이는 ideation Round 3 결정의 위치 부분이 진화한
것으로, vault에는 hub note + 출판물 promotion만 두고, 코드 / SQLite / R2 운영 /
ADR / PRD / HLD는 외부 repo에서 운영하는 형태가 되었다.

## Decision

이 repo (`k-world-monitor`)는 second-brain ideation의 외부 코드 repo로 운영
한다.

- 코드 / 마이그레이션 / SQLite (`research.db`) / 마크다운 vault(이 repo의
  Markdown publication 생성 단계)는 모두 이 repo 안에 둔다.
- second-brain 측에는 hub note + 외부 repo 등록 (`_System/AI/EXTERNAL_SOURCES.yaml`)
  까지만 두고, canonical PRD / HLD / ADR은 본 repo에 둔다 (second-brain
  `SOURCE_OF_TRUTH.md`의 "Project PRD/HLD/ADR → Project repo" 정책 준수).
- 이 결정은 ideation Round 3 위치 결정(vault 내 형)을 supersede 하는 것이
  아니라 실행 시점에서의 운영 형태 진화로 본다 — ideation 결정의 객체 모델 /
  storage 분담 / cite check 정책 등은 모두 그대로 유지된다.
- second-brain 측 hub note 생성과 `EXTERNAL_SOURCES.yaml` 등록은 별도 작업
  (사용자 수행). 본 repo는 외부 repo로서의 책임만 진다.

## Rationale

- 코드 산출물 + CI + 외부 의존성(R2 / Anthropic SDK) 운영은 외부 repo가
  자연스럽다 (second-brain `CLAUDE.md` "외부 repo 형 vs vault 내 형" 가이드
  중 "코드 산출물 중심·CI 필요 → 외부 repo 형").
- vault 내 `_store/research.db` 운영은 vault git 히스토리에 binary diff 누적
  위험 + 마이그레이션 history 충돌 위험.
- ideation Round 3 위치 결정의 핵심 의도(canonical PRD/HLD/ADR을 어디 두느냐)는
  이 결정으로도 유지된다 — 외부 repo `docs/`에 canonical.

## Consequences

- 긍정:
  - boilerplate(`alxdr3k/boilerplate`) Tier 2 채택으로 invariant tracking
    (ADR-0002), per-file Q/DEC, glossary, sync overlay 등 인프라 자동 도입
  - SQLite / R2 binary가 vault git에 들어가지 않음
  - 본 repo는 main 직접 push 허용 (global policy의 actwyn/concluv/boilerplate/
    my-skill/devdeck/k-world-monitor 군) — 빠른 이터레이션
- 부정:
  - second-brain hub note + `EXTERNAL_SOURCES.yaml` 등록 작업이 별도 필요
    (사용자 수행)
  - publication artifact가 vault에서 발행될 경우 vault ↔ 외부 repo 동기화 정책
    필요 (PUB-1A 단계에서 결정)
- Follow-ups:
  - second-brain 측 hub note 생성 (사용자 수행, 본 repo 외부)
  - second-brain 측 `EXTERNAL_SOURCES.yaml`에 `repo_k_world_monitor` entry 추가
  - PUB-1A 단계에서 vault publication 동기화 정책 결정 (DEC 또는 ADR)
