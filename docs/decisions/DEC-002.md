# DEC-002: second-brain ideation settled (2)(3)(4)의 본 repo 적용 boundary

## Context

second-brain ideation `research-content-pipeline-architecture` Round 3 "Agreed /
settled" 14항 중 (2)(3)(4)는 vault 측 결정으로, 본 repo (외부 코드 repo)에서는
직접 적용 대상이 아니다. 그러나 정합성 검증에서 "본 repo가 이 결정들을 어떻게
다루는가"가 명시되지 않으면 누락으로 비친다. 이 DEC는 그 boundary를 lock한다.

ideation 원문 settled 항목:

- (2) **위치**:
  - 출판: `05. Projects/세계경제-시나리오-콘텐츠-파이프라인/Publications/`
  - 리서치: `06. Areas/세계경제-지정학-감염병-모니터링/`
  - Bulk store: 위 area 산하 `_store/research.db` + `_store/*.jsonl`
- (3) **CLAUDE.md 수정**: Projects의 external-repo 의무 지시문 제거, "외부 repo
  사용 시" 옵션으로 표현
- (4) **스키마**: `type` enum 그대로 + `research:{object_type, schema_version}`
  직교 필드, JSON Schema if/then/else로 conditional 검증

## Decision

본 repo는 위 settled 항목 (2)(3)(4)를 다음과 같이 처리한다.

| Settled # | 본 repo 적용 |
|---|---|
| (2) 위치 | DEC-001로 외부 repo 형 진화 결정 — vault 위치는 second-brain 운영자가 hub note + EXTERNAL_SOURCES.yaml 등록으로 처리. 본 repo는 외부 repo로서 PRD/HLD/ADR/runbook을 canonical로 보유 |
| (3) CLAUDE.md 수정 | second-brain repo의 `CLAUDE.md` 본문에 이미 "vault 내 형 vs 외부 repo 형" 분기로 반영 완료 (확인: second-brain `CLAUDE.md` "외부 repo 형" 섹션). 본 repo는 별도 액션 없음 |
| (4) `research:{object_type, schema_version}` 직교 필드 | second-brain vault `_System/Schemas/NOTE_SCHEMA.md`의 type enum 정책. 본 repo는 vault에 노트를 자동 생성하지 않으므로 적용 대상 아님. 본 repo의 SQLite 스키마(ADR-0004)는 자체 컬럼 체계를 가짐. 다만 향후 본 repo가 vault로 promotion artifact를 export할 경우 그 promotion artifact는 vault 스키마(직교 필드)를 준수해야 함. 이 export 정책은 PUB-1A 단계에서 별도 DEC로 결정 |

## Rationale

- ideation 결정이 "vault 내 형"을 전제로 했으나 실행 시점에서 외부 repo 형으로
  진화 (DEC-001). 그 진화로 (2)(3)(4)는 vault 측 책임이 됨.
- 본 repo가 vault 결정에 의존하는 것이 아니라, vault export boundary에서만
  접점이 발생함을 명시.
- 정합성 검증에서 settled 항목이 "본 repo에 매핑되지 않음"을 분류 없이
  남기면 missing으로 잘못 분류될 위험.

## Consequences

- 긍정:
  - settled (2)(3)(4)에 대한 본 repo 입장이 명확
  - PUB-1A 단계의 vault export 정책 결정 위치가 미리 정해짐
- 부정:
  - vault 운영자(사용자)는 second-brain 측 hub note + `EXTERNAL_SOURCES.yaml`
    등록 작업을 별도 수행해야 함 (이 repo 외부)
- Follow-ups:
  - PUB-1A 단계에서 vault export 정책 DEC 또는 ADR 작성 (시기: P0-M6 진입 직전)
  - second-brain 측 hub note 생성 확인 후 본 DEC 본문에 anchor 추가
