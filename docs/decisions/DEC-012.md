# DEC-012: Editorial Quality Rubric — AC-036~042

## Context

GPT 메타 리뷰 (2026-05-11) 의 4 순위 병목 지적:

> 현재 AC 들은 trace, schema, category, cross-post anchor, build gate 같은
> 기술 품질을 잘 잡습니다. 하지만 "좋은 글인가?" 를 판단하는 editorial gate
> 는 아직 약합니다. 필요한 것은 별도 Content Quality Rubric 입니다.

본 repo 의 비판적 리뷰 = **수용**. AC-001~035 의 technical gate 만으로는
"Zod schema 통과한 평범한 글" 이 발행 가능 — editorial gate 신규.

## Decision

PUB-1A.5 accept 시 **운영자 manual verify checklist 7 종 의무** — AC-036
~ AC-042 신규.

### Rubric (AC-036 ~ AC-042)

| AC | Rubric | 검증 방법 |
|---|---|---|
| **AC-036** (CQ-001) | thesis 가 **한 문장** 으로 명확하게 진술되어 있다 | 운영자 read-aloud — 10초 안에 thesis 1문장 정확히 인용 가능 |
| **AC-037** (CQ-002) | 독자가 글을 통해 얻는 **새로운 관점** 이 있다 (단순 source summary 가 아님) | 운영자 self-check — "내가 이 글을 안 쓰고 source 만 읽었으면 같은 결론 도달했을까?" |
| **AC-038** (CQ-003) | supporting evidence 와 opposing evidence 가 **둘 다** 본문에 등장한다 | ADR-0027 minimum coverage (supporting ≥ 3 / opposing ≥ 2) 자동 충족 + 본문에서 운영자 manual 확인 (예 "그러나 ..." / "반론은 ..." section 존재) |
| **AC-039** (CQ-004) | target 별 upside / downside 가 **분리** 되어 있다 (REQ-021 impact_targets / impact_direction_by_target 이 본문에 visible) | DEC-011 의 10 target × impact_direction 표 같은 분해가 본문에 명시 |
| **AC-040** (CQ-005) | monitoring signal 이 **구체적** 이다 (modal verb 가 아니라 측정 가능 indicator) | 예 "주시할 만하다" X / "월별 미분양 추이 5만 호 초과 시" O |
| **AC-041** (CQ-006) | 과장된 forecast language 없다 (단정적 prediction X, 확률 / 조건 / horizon 명시) | 예 "폭락한다" X / "X 조건이 발생하면 1~3년 내 가격 -30% 이상 시나리오 가능성 ↑" O |
| **AC-042** (CQ-007) | correction 가능성 / 불확실성이 **명시** 되어 있다 (글 말미 또는 별도 section) | "본 thesis 의 retraction trigger" 또는 "이 글이 틀릴 수 있는 시나리오" section 존재 |

### 운영자 manual verify (v0)

- PUB-1A.5 의 publication preflight gate 진입 시 운영자가 위 7 AC 를 본문
  대조 확인.
- 7 종 모두 pass 해야 publication state = `live` 전환 가능.
- 미달 시 ContentDraft state = `reviewing` 유지 (DEC-005 manual correction
  approve workflow 와 정합).

### v1+ LLM judge 옵션 (별도 Q)

- v1+ 에 LLM judge (Tier 3 GPT-5 nano + Haiku 4.5 cross-vendor review,
  ADR-0023 INV-0023-4) 가 7 AC propose → 운영자 confirm 으로 자동화 가능.
- v0 는 manual only — 운영자 thesis 판단 + editorial 감각 형성 단계.

### Editorial gate vs Technical gate 분리 (ADR-0015 extends_scope)

- **Technical gate** (ADR-0015 cite check 5+1) = stale / retracted / horizon
  / unit / overclaim + access_intervention block + (v1+) one-sided thesis
  warning. **자동** + build-time (AC-035) / runtime (AC-013).
- **Editorial gate** (본 DEC, AC-036~042) = 글 자체의 콘텐츠 품질. **manual
  v0** / LLM judge v1+ 옵션.
- 두 gate 모두 publication preflight 의무.

### Editorial gate 의 Astro Zod schema 통합 (선택, v1+)

- v0: ContentDraft frontmatter 에 `editorial_quality_rubric_passed` boolean
  필드 추가 (운영자 lock). false 또는 missing 시 Astro Zod schema build
  fail (ADR-0022 INV-0022-3 의 build-time gate 확장).
- v0 운영자 manual setting — `pipeline publish` 의무 step 으로.

## Rationale

- GPT 메타 리뷰 4순위 병목 (editorial gate 부족) 직접 대응
- 기존 cite check 5+1 (ADR-0015) 의 technical gate 와 명확 분리 — 둘 다
  의무
- 운영자 manual (v0) → LLM judge (v1+) 단계화 — 1인 운영 부담 안에서 시작
- ADR-0027 evidence_role minimum coverage 와 AC-038 자연 정합 — 분리된
  결정이지만 서로 보완
- AC-036~042 가 "글 자체" 판단 framework 제공 — 운영자가 매번 "이 글이
  좋은가?" 를 7 dimension 으로 분해해 verify

## Consequences

- 긍정:
  - publication preflight 에 editorial gate 추가 → Zod schema 통과한 평범
    한 글 발행 차단
  - thesis / supporting / opposing / target 비대칭 / monitoring / forecast
    tone / correction 가능성 7 dimension framework → 운영자 thesis 판단력
    구조화
  - v0 manual + v1+ LLM judge 단계화 → 운영 부담 점진 완화
  - Astro Zod schema 통합 (v0 boolean flag) → build-time enforce 가능
- 부정:
  - PUB-1A.5 accept 의 운영자 workload +7 AC verify (1 글당 5~15분 추가)
  - editorial gate 의 7 AC 가 v0 에서 manual 만 — flaky / subjective 위험
    (단 ContentDraft state `reviewing` 으로 reject 가능)
  - LLM judge 자동화는 v1+ — v0 운영자 attention cost 흡수 의무
- Follow-ups:
  - AC-036 ~ AC-042 신규 (`docs/06_ACCEPTANCE_TESTS.md`)
  - TEST-036 ~ TEST-042 신규 (manual verify checklist, 자동 TEST 는 v1+
    LLM judge 도입 시점)
  - ContentDraft frontmatter `editorial_quality_rubric_passed` 필드 추가
    + Astro Zod schema 확장 (PUB-1A.4 / PUB-1A.1 갱신)
  - PUB-1A.5 운영자 verify checklist UI (`pipeline publish --editorial-
    verify`)
  - REQ-027 (PRD) 의 AC mapping 에 AC-036~042 추가
  - 신규 Q — LLM judge 자동화 v1+ 시점 (별도)
