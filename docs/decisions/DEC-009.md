# DEC-009: v0 turn-key first publication category = 경제 (economy)

## Context

DEC-004로 v0 카테고리를 4 메타(정책 / 경제 / 사회 / 대중문화)로 lock.
DEC-005로 v0 turn-key publish scope를 blog_long 1개로 lock. PUB-1A.5
(첫 publication MVP gate)이 어느 카테고리로 첫 발행할지 결정 필요.

## Decision

v0 turn-key MVP gate의 첫 publication 카테고리를 **경제 (economy)** 로
lock한다.

이 결정이 영향을 주는 것:

1. **Tier A source seed (Q-021)**: 30개 중 경제 12개 source 우선 부트스트랩.
   상세 list는 `docs/research/source-seed-list-2026-05.md` 참조. source_perspective
   분포 (risk_observer ≤ 50%, opportunity_observer ≥ 25%, neutral ≥ 15%)
   강제 — 경제 카테고리 안에서 충족.
2. **Dossier composer prompt**: 경제 도메인 knowledge(macro_finance /
   trade_supply_chain / energy_commodities / digital_assets 4 subtopic)
   우선. Sonnet 4.6 escalate domain trigger에 "거시"가 이미 specialty
   (DEC-008) 로 lock되어 있으므로 Scenario composer는 사실상 Sonnet
   default.
3. **Scenario validator + Thesis composer**: 경제 도메인의 transmission_channel
   (energy / credit / fx / policy / technology / information / population)
   우선 적용. impact_targets은 region(global / US / China / EU / Korea
   / Japan / ASEAN) 기반.
4. **자체 사이트 첫 publication URL**: `https://<self-site>/posts/<slug>` —
   ADR-0022 INV-0022-2 canonical anchor. blog_long 1건 (DEC-005).

나머지 3 카테고리(정책 / 사회 / 대중문화)는 v0 진행 중 누적 도입 —
PUB-1A.5 첫 발행 이후 매주~격주 cadence 로 추가 (Tier A seed 정책 7 +
사회 6 + 대중문화 5 점진 등록).

## Rationale

- 경제 카테고리는 1인 콘텐츠 발행자의 일반적 strong domain (4축 병렬
  분석이 가장 정합)
- Tier A source가 가장 풍부하고 access_method 가 API/RSS 자동 수집 가능한
  공식 endpoint 가 많음 (Fed FRED API, IMF API, World Bank Open Data,
  BIS warehouse, KOSIS Open API 등)
- source_perspective 분포 균형 가장 쉽게 충족 (Fed/BOK/ECB risk_observer
  + IMF/OECD/World Bank opportunity_observer + KOSIS/BLS/EIA/IEA/IGC
  neutral)
- macro_finance 는 DEC-008 의 specialty 도메인이므로 Sonnet escalate
  자동 — quality 안정성 우위
- bidirectional framing(ADR-0019) 4축이 경제 도메인에서 자연스럽게 분기
  (예 인플레이션 risk / 생산성 opportunity / 회복탄력성 supply chain
  / asymmetric impact 신흥국 vs 선진국)

## Consequences

- 긍정:
  - PUB-1A.5 첫 발행까지의 시간 박싱 가능 (Tier A seed 12개 + 1 Dossier
    + 1 Scenario + 1 Thesis + 1 ContentDraft + 1 자체 사이트 publication)
  - Sonnet escalate 도메인 trigger 와 정합 (specialty 거시 = 자동
    Sonnet)
  - 도메인 knowledge prompt 우선 작성 — extract / dossier / scenario /
    thesis 모두 경제에 튜닝
- 부정:
  - 나머지 3 카테고리 진입이 v0 turn-key 이후로 지연 — 4축 병렬 다양성은
    v0 진행 중 누적
  - 경제 카테고리에 도메인 specificity가 누적될수록 정책/사회/대중문화
    onboarding 마찰 증가 (prompt 재튜닝 필요)
  - 경제 도메인의 외부 cross-post audience (Substack 경제 뉴스레터 /
    YouTube 거시 채널 / X economist) 와의 경쟁 격화 — 자체 사이트의
    cite anchor canonical + bidirectional framing 차별화에 의존
- Follow-ups:
  - Slice `INFRA-1A.6` (Tier A seed): 경제 12 source 우선 등록 — `docs/
    research/source-seed-list-2026-05.md` 의 경제 섹션 그대로
  - Slice `PUB-1A.5` (첫 publication): 경제 카테고리 1 Dossier → 1
    Scenario → 1 Thesis → 1 ContentDraft(blog_long) → 1 publication
  - 정책/사회/대중문화 카테고리 onboarding 시점 별도 결정 (PUB-1A.5
    이후 매주~격주 cadence 권고, 운영자 capacity에 따라)
  - 첫 publication의 thesis가 어느 시나리오 axis(risk_observer /
    opportunity_observer / neutral / asymmetric)를 가장 강하게 가져갈지
    는 PUB-1A.5 진입 시점에 운영자 선택
