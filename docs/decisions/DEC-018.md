# DEC-018: RSS/XML 파서 선택 — fast-xml-parser + XXE 비활성화

## Context

Discovery worker (INFRA-1B.2a) 가 RSS / Atom XML 을 파싱할 때 XXE (XML External
Entity) 취약점이 있는 파서를 사용하면 원격 파일 읽기, SSRF, DoS 가 가능하다.
Opus 리뷰 Q4: "RSS 는 XML 이다. 파서를 XXE 비활성화 기본값으로 골라야 한다."

파서 선택지:
- `xml2js` — XXE 기본 활성화 (취약). 사용 금지.
- `fast-xml-parser` v4+ — `processEntities: false` 옵션으로 XXE 완전 차단. 선택.
- `sax` (bun 내장) — 스트리밍 파서, 저수준 API 부담. v1+ 검토.

## Decision

RSS / Atom XML 파싱에 `fast-xml-parser` v4+ 를 사용한다.

필수 설정:
```typescript
import { XMLParser } from "fast-xml-parser";

export const RSS_PARSER = new XMLParser({
  processEntities: false,     // XXE 비활성화 (핵심)
  ignoreAttributes: false,
  attributeNamePrefix: "@",
  allowBooleanAttributes: true,
});
```

이 설정 객체는 `src/discovery/parse/xml-safe.ts` 에서 singleton 으로 export 한다.
코드베이스 내 `new XMLParser()` 직접 호출(설정 없이) 금지 — lint 또는 convention.

## Rationale

- `fast-xml-parser` v4 는 `processEntities: false` 로 XXE 공격을 완전 차단
- Bun/Node 생태계에서 활발히 유지되고 있어 취약점 패치가 빠름
- singleton export 패턴으로 설정 누락을 구조적으로 방지

## Consequences

- 긍정: XXE 공격 차단 보장
- 부정: `fast-xml-parser` 의존성 추가 (경량 — gzip 20KB)
- Follow-ups: Slice `INFRA-1B.2a` 에서 `xml-safe.ts` 구현 + 단위 테스트 (XXE payload 차단 포함)
