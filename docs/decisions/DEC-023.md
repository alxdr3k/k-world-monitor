# DEC-023: Q-051 Round 1 routing default 5항 lock

## Context

Q-051 (Research App UI surface) 는 본 repo 에서 PR #33 (`claude/q-049-q-050
-resolution`) 안에서 round 2 web pivot 후 #6 UI stack 만 DEC-022 lock,
나머지 Round 1 routing default 5 항목은 status: open 으로 남았음
(round 17 Codex finding 후 partial resolution 명시).

본 DEC 는 GPT 33 review (2026-05-15) + 사용자 동의로 5 항목 일괄 lock.
Q-051 status: resolved 복귀.

## Decision (lock)

### #1 — Routing 우선순위 default (`flag 없고 active context 도 없을 때`)

**채택**: **API/CLI layer = error + Web UI layer = explicit one-tap
new_session 버튼**.

```
[API/CLI]
POST /api/ask 또는 `pipeline ask "<message>"` 호출 시
  flag 없음 (--session / --round / --new-session / --new-round /
            --branch-from / --fork-session / --answer-only 미지정)
  + operator_context.active_session_id IS NULL
  + operator_context.active_round_id   IS NULL
→ HTTP 400 / CLI exit 1 + error message:

  No active session/round.
  Choose one:
    --new-session
    --session <id>
    --round <id>

[Web UI]
/ops/ask 진입 시 active context 가 NULL 인 경우 "새 세션으로 시작" 버튼을
default highlight. 사용자가 submit 하면 명시적으로 route = new_session 으로
API 호출.
```

**근거**:
- ResearchSession 은 단순 chat thread 가 아니라 ExplorationRound /
  RoundSynthesis / Dossier / Scenario lineage 의 상위 컨테이너 → 자동
  새 session 누적 시 lineage trace 흐려짐
- API contract 안전성 (governance) + UX 친화성 (모바일 1-tap) 분리 lock
- Q-051 Q-H "AccessIntervention 모바일 1-tap" 결정과 호환 — UX 마찰 X

### #2 — AI intent classifier 활성 시점

**채택**: **(ii) RESEARCH-1A.2 이후 advisory-only 도입**. RESEARCH-1A.1
안에서는 explicit route selector dropdown 만.

**근거**:
- classifier 가 advisory only 여도 모든 ask 마다 추가 LLM call + run_ledger
  row + prompt injection handling + audit field 필요
- cold start 에서 confidence calibration 데이터 부재 — 초기 confidence
  값이 실제 정확도와 무관
- Q-051 의 우선순위는 "질문이 올바른 session/round 에 기록" — 자동 route
  추천 아님
- RESEARCH-1A.1 = session/round/turn persistence + explicit route
  selector 안정화, RESEARCH-1A.2 = classifier 추가 phasing

### #3 — Non-sensitive prompt threshold (ENV tuning)

**채택**: **(p) 표시만, 자동 변경 없음** + `KWM_AUTO_ROUTE_CONFIDENCE_MIN`
ENV variable **reserved but inactive** (후속 DEC 까지 enforcement 없음).

```
[Classifier suggestion UI]
ask bar 옆 chip 표시:
  "이 질문은 new_round 일 가능성이 높습니다 (confidence 0.82)"

Action:
  classifier 추천이 어떤 route 든 (sensitive vs non-sensitive 무관)
  사용자가 dropdown / chip-tap 으로 명시 선택해야 적용.
  자동 route change 없음.

ENV:
  KWM_AUTO_ROUTE_CONFIDENCE_MIN — reserved placeholder. v1+ 별도 DEC 에서
  enforcement 도입 시점에 의미. P0-M6 ~ RESEARCH-1A.2 까지 inactive.
```

**근거**:
- classifier confidence 가 calibration 안 된 상태에서 auto-route 시
  silent drift 위험 (잘못 routed turn → 잘못 lineage)
- new_round 도 잘못 적용되면 RoundSynthesis + next_round_plan 끊김
- answer_only 잘못 적용되면 중요 탐색이 evidence ledger 에 누락
- ENV variable 만 reserved → v1+ 운영 데이터 누적 후 별도 DEC 발급 시
  enforcement 도입 가능 (P0-M6 lock 안 영향 X)

### #4 — Fork session 시 metadata 승계 범위

**채택**: **`fork_kind` 별 다른 inheritance 정책 (Q-051 Q-J 와 일관)**.

```
fork_kind = side_exploration → α (fresh intent only)
  inherit:
    forked_from_session_id
    fork_kind
    title (운영자 명시)
  fresh:
    initial_intent
    editorial_intent_id  → NULL
    mode                 → fresh choice (default exploratory)

fork_kind = derivative → β (soft inherit mode + editorial_intent_id)
  inherit:
    forked_from_session_id
    fork_kind
    title (운영자 명시)
    mode                 (parent 와 동일)
    editorial_intent_id  (parent 의 editorial intent reuse — 별도 publication
                          후보로서 동일 intent 발전)
  fresh:
    initial_intent       (새 question 기준)
    Dossier              (promoted_claim_ids[] NOT inherited)

Disallowed by default (γ deep inherit):
  Dossier.promoted_claim_ids[] deep copy
  Claim node references inheritance
  → 별도 사용자 명시 + 별도 DEC 없이는 금지
```

**근거**:
- fork = "새 lineage" 시작 → parent Dossier promoted claims deep inherit
  시 selection bias 강화 위험
- `side_exploration` 은 parent hypothesis 오염 없이 새 가설 탐색 의도 →
  fresh intent 강제
- `derivative` 는 별도 publication 후보 → parent intent (audience / tone
  / bidirectional_weight_intent) 유지하면서 새 thesis 발전 → soft inherit
- Dossier evidence 는 RoundContextPack 에서 reference 로 조회 가능 →
  canonical evidence 로 자동 복제 불필요

### #5 — `.kwm/` CLI wrapper 디렉토리 위치

**채택**: **deferred 유지**. 필요 시 XDG Base Directory spec (z) 채택.

```
Status:  deferred (P1+ CLI wrapper 도입 시점에 결정).
If/when needed:
  config:  $XDG_CONFIG_HOME/kwm/config.json
           fallback: ~/.config/kwm/config.json
  data:    $XDG_DATA_HOME/kwm/
           fallback: ~/.local/share/kwm/
Not chosen:
  repo root .kwm/  — multi-checkout ambiguity + git accidental staging
                     위험. operator context 는 사용자/환경 level.
  $HOME/.kwm/      — XDG 미준수.
```

**근거**:
- P0-M6 ~ RESEARCH-1A.1 critical path 가 아님 (Web /ops 가 primary,
  CLI wrapper P1+ secondary)
- operator_context 는 DB (SQLite) 에 canonical 저장 — CLI local state 는
  convenience cache 일 뿐
- 지금 위치 lock 하면 over-engineering. 도입 필요 시점에 다시 결정

## Consequences

### Positive

- **lineage integrity** — no-context error 가 orphan session 누적 방지.
  Dossier / Scenario lineage trace 정합성 ↑
- **2주 P0-M6 lock 안전** — classifier deferred (RESEARCH-1A.2) 로
  P0-M6 안에서 LLM-side 부담 0. simple drop-down UX 안정화 우선
- **Selection bias 방지** — fork inheritance 가 `fork_kind` 별 분리 →
  side_exploration 의 다양성 + derivative 의 publication intent 연속성
  양립
- **ENV reservation pattern** — `KWM_AUTO_ROUTE_CONFIDENCE_MIN` reserved
  but inactive — v1+ 운영 데이터 누적 후 enforcement 도입 가능 path
  유지

### Negative / 주의

- **API/CLI error 가 모바일 UX 마찰** — mitigated: Web UI 에서 "새 세션
  시작" 버튼 default highlight 로 1-tap 보장 (Q-051 Q-H 호환)
- **classifier 도입 RESEARCH-1A.2 phasing** — initial UX (explicit
  dropdown) 검증 후 classifier 추가 → release sequence 1단계 추가.
  P0-M6 critical path 외이므로 영향 작음
- **ENV variable placeholder** — reserved 명세만 정의, 실제 enforcement
  없음. 향후 v1+ DEC 발급 시점에 활성화 의무 — 잊혀짐 위험 있으나
  RESEARCH-1A.2 슬라이스 spec 안 명시
