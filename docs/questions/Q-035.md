# Q-035: Google Gemini 사용 scope 확장 시점 (v1+ 메인/리뷰)

## Context

ADR-0023 + DEC-010 (2026-05-11) 으로 v0 Google (Gemini) 사용 scope 를
탐색 보조 (Google Search grounding) + Tier 3 비용효율 우위 시 fallback 으로
한정. 메인 generator + cross-vendor reviewer 사용 절대 금지.

이유: Gemini 의 메인 사용 신뢰도 검증 불충분 (사용자 결정). 그러나 운영
데이터 누적 후 SPIKE 입증 시 v1+ 확장 검토 여지.

## Discussion

v1+ 확장 후보:

1. **Tier 1~2 default vendor 후보 추가** — 운영 200+ 호출 데이터 누적 +
   SPIKE 로 Gemini Pro/Ultra 가 GPT-5.5 Pro standard / GPT-5 mini 대비
   quality > 5% 우위 입증 시 domain override 룰 안에서 사용 가능 추가.
2. **한국어 long-context 작업** — 현재 Sonnet 4.6 standard 가 한국어
   long-context 의 domain override. Gemini 2.5 Pro 의 한국어 quality
   비교 SPIKE 결과에 따라 override 후보 추가.
3. **Cross-vendor review 3-vendor 확장** — 현재 cross-vendor 쌍 = OpenAI/
   Anthropic 2-vendor. v1+ preflight 단계에 한해 Google 추가하면 3-vendor
   review (multi-judge consensus). 단 cost ↑ 부담 — 운영 cost ceiling
   재산정 필요.
4. **Discovery layer 확장** — 현재 Search grounding 만 사용. v1+ 에 Gemini
   의 web 검색 + multi-modal (이미지/PDF) 기능을 Discovery 확장에 통합.

trigger:
- 운영 200+ LLM 호출 누적 (운영 시작 후 약 2~4주 예상)
- vendor 별 cache_hit_rate / cost_per_promoted_claim / faithfulness_rate
  실측 비교 SPIKE
- Google 의 quality benchmark / price update tracking (Artificial
  Analysis 등)

## Resolution

resolved 시 → DEC-### / ADR-####. v1+ EXTR-1B 트랙 진입 시점에 lock.
SPIKE 결과 (vendor 비교) 누적 후 별도 ADR (ADR-0023 supersede) 또는
DEC (DEC-010 확장) 발급.
