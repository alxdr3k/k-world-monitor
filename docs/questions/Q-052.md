# Q-052: main branch protection 3중 정책 충돌 reconciliation

## Context

2026-05-15 Claude 12-layer adversarial review (`docs/10_PROJECT_RETROSPECTIVE.md`
의 `## Adversarial Review — 2026-05-15` section, L9-A finding) 가 발견.
GitHub API 실측:

```
$ gh api repos/alxdr3k/k-world-monitor/branches/main/protection
{"message":"Branch not protected","status":"404"}
```

세 layer 의 단언이 서로 다름:

| 문서 / 정책 | 단언 | 실제 fact |
|---|---|---|
| docs/01_PRD.md:208-212 | "main branch protection: PR-only (DEC-020 Q-048 resolution, 2026-05-13). required check = .github/workflows/ci.yml" | ✗ Branch not protected |
| docs/04_IMPLEMENTATION_PLAN.md TRACE-044 | "CI required check 등록 (ci.yml) + main push 정책 해제 (PR-only)" | ✗ admin task 미실행 |
| CLAUDE.md (user global) "Git Branch Policy" | "개발 중 프로젝트 (... k-world-monitor ...) main 에 직접 커밋/push 허용" | ✓ 실제 GH state 와 정합 |

이 결정 없이는 향후 모든 doc drift / governance baseline 이 흔들림.

## Discussion

- (a) DEC-020 우선 — multi-collaborator 안전성 ↑, 단일 운영자 작업 overhead ↑
- (b) CLAUDE.md 우선 — 실제 운영 정합, 외부 collaborator 진입 시 reflow 의무

## Resolution

DEC-024 D1 resolves this Q.

**Decision**: DEC-020 우선. **PR-only main branch policy** 가 canonical target.

- `docs/01_PRD.md:208-212` + DEC-020 Q-048 의 "PR-only" canonical 유지
- `CLAUDE.md` global 의 k-world-monitor 직접 push 예외는 운영자 admin task 로 제거 또는 break-glass 만 허용 으로 축소
- 실제 GitHub branch protection 등록 = 운영자 admin task
- 미등록 동안 docs 는 "policy target: PR-only; admin task pending" 으로 표기

Break-glass 절차 (direct push 예외 허용 사례):
- security credential leak 즉시 제거
- broken main 이 모든 PR check 차단 시 복구
- operator explicit emergency action

Break-glass 후 의무: retrospective note + follow-up PR or repair commit.
