# Q-043: AC-044 (Dossier evidence_role minimum coverage, ADR-0027) 가 `06_ACCEPTANCE_TESTS.md` 에 정식 정의되어 있는가

## Context

본 Q 는 2026-05-13 comprehensive review 의 사용자 결정 필요 항목 #2 이다.

ADR-0027 (`docs/adr/0027-claim-level-evidence-role-classification.md`) 은
Dossier 안의 promoted claim 을 supporting / opposing / monitoring 3 가지
evidence_role 로 분류하는 invariant 를 lock 한다. P0-M6 turn-key MVP
gate (PUB-1A.5) 는 발행 직전 operator manual verify 의무 안에 본 invariant
의 minimum coverage 검증을 포함한다.

`04_IMPLEMENTATION_PLAN.md:83` 에서는:

> AC-044 (evidence_role minimum coverage, ADR-0027)

로 게이트 row 가 등록되어 있다. 그러나 `06_ACCEPTANCE_TESTS.md` AC 표
검사 결과 AC-### row 가 AC-001 ~ AC-042 까지 43 row 인 것으로 확인됐다
(grep 결과). AC-043 / AC-044 의 정식 정의 row 존재 여부가 본 Q 의 핵심
확인 사항이다.

본 Q 가 resolved 되지 않으면 PUB-1A.5 게이트 검증 시점에 "AC-044 어떤
조건이 충족되어야 통과인가" 가 ambiguous 하다.

## Discussion

ADR-0027 본문의 invariant 와 AC-### 의 차이:

- ADR INV: architecture-level lock, 의미 단위.
- AC: measurement-ready gate, given/when/then 형식 + TEST-### 위치.

ADR-0027 의 INV 가 직접 게이트 검증에 쓰일 수 있는 형태인지, 아니면 별도
AC-044 row 가 필요한지는 본 Q 가 결정한다. PR #25 후속 review 에서도
"supporting ≥3 / opposing ≥2 / monitoring ≥3 + operator_lock" 의 수치는
명시적으로 lock 됐으나 측정 방법 (Dossier 단일 발행 직전 검증 vs Dossier
build pipeline 안에서 자동 검증) 은 lock 안 됨.

기본 권고 (옵션 a) 의 작업 순서:

1. `06_ACCEPTANCE_TESTS.md` AC 표에 AC-043, AC-044 row 추가 (AC-043 은
   ADR-0027 의 evidence_role schema lock, AC-044 는 minimum coverage 의무).
2. `09_TRACEABILITY_MATRIX.md` 에 ADR-0027 → AC-043/044 trace row 추가.
3. TEST-043, TEST-044 위치 결정 (구현 시점은 PUB-1A.2 cite check 슬라이스 또는
   AGG-1A.1 dossier 합성 슬라이스).
4. REQ-### 신규 row 가 필요한지 결정 (현재 REQ-001~027 — REQ-028 신규 vs
   REQ-027 본문 확장).

## 사용자 결정 필요 항목

1. AC-043, AC-044 를 정식 AC row 로 추가할 것인가, 아니면 ADR-0027
   internal invariant 로만 유지하고 P0-M6 게이트 reference 를 제거할
   것인가
2. 추가 시 REQ-028 신규 row 가 필요한가, 아니면 REQ-027 본문 확장으로
   충분한가
3. TEST-043 / TEST-044 의 측정 위치 — Dossier 합성 (AGG-1A.1) 안에 자동
   검증을 넣을 것인가, 아니면 PUB-1A.5 발행 직전 operator manual verify
   로 둘 것인가
