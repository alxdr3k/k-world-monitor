# Q-026: Vault sync trigger

## Context

DEC-001로 외부 repo 형 운영을 확정. vault에는 ProjectHub + Area hub +
promoted artifact만 둔다. 어느 시점에 vault sync가 trigger되는지 결정 필요.

## Discussion

후보:
- (a) publication 시점 자동 — publish 명령 성공 시 vault에 publication record
  + 인용된 promoted claim/dossier/scenario/thesis가 자동 sync
- (b) manual only — `pipeline vault-sync` 명령으로 운영자가 명시 실행
- (c) hybrid — publication은 자동, 나머지(dossier/scenario/thesis promotion)는
  manual

권고: (c) hybrid.
- Publication: 자동 sync (publish 시점에 vault에 record 생성, 인용된 promoted
  claim/dossier/scenario/thesis 함께)
- 그 외 promoted 객체: manual `pipeline vault-sync --dossier <id>` / `--scenario
  <id>` / `--thesis <id>`

## Resolution

**Resolved by DEC-006** (2026-05-11) — ADR-0022 자체 사이트 stack 결정 후
vault publications/ 가 자체 사이트의 single source로 직결됨에 따라 Q-026의
hybrid 권고는 git push 단일 trigger 로 단순화. Cloudflare Pages 가 git
integration 으로 publications/ 변경분을 자동 build + deploy. vault 외
디렉토리(dossiers / scenarios / theses / content_drafts / promoted_claims)는
자체 사이트 비노출(ADR-0022 INV-0022-4)이므로 별도 sync 명령 v0에서 불필요.
v1+ 외부 플랫폼 auto cross-post(Q-033) 도입 시 별도 sync 워크플로 재검토.
