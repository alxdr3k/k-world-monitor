# Q-004: SQLite vs vault jsonl 책임 분담

## Context

ideation Round 3 Q4 — second-brain vault는 `_System/Indexes/*.jsonl`로
cross-area index를 유지한다. 이번 파이프라인의 SQLite와의 경계가 모호하다.

이 repo(k-world-monitor)는 외부 코드 repo로 분리되었으므로 vault와의 직접
공유는 없다. 그러나 future가 second-brain vault에 promoted artifact를 export
하거나 vault index를 import할 때 책임 boundary를 정해야 한다.

## Discussion

후보 분담:

- A: research-internal(claim/snapshot/edge/scenario_revisions/run)은 SQLite,
  vault-wide cross-area index(thread_index 등)는 jsonl 유지. k-world-monitor
  repo는 SQLite만 보유. promoted artifact를 vault로 export 시 vault의 jsonl
  형식으로 변환.
- B: vault jsonl 폐기 + 모든 index를 SQLite로 통일. vault는 단순 markdown 저장
  소가 됨 — 큰 영향 범위.
- C: 두 시스템 동기화 — 중복 저장, drift 위험.

권고: A. boundary 명확, vault 영향 최소, 이 repo는 단일 store.
