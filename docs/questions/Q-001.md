# Q-001: scenario horizon enum 정의

## Context

ADR-0008의 horizon mismatch cite check와 ADR-0009의 scenario validate가 활성화
되려면 scenario.horizon의 enum 값이 lock되어야 한다. ideation Round 3에서
보류된 4개 Open Question 중 하나(Q1).

후보 분류:

- (a) 시점 기반 4-tier: tactical (≤1Q) / near (≤1Y) / mid (≤5Y) / generational (>5Y)
- (b) 단순 3-tier: short / mid / long
- (c) 자유 텍스트 + 시작/끝 date range
- (d) 비교 가능한 month 단위 정수 (예: 3, 12, 60, 120)

## Discussion

- 거시 콘텐츠 특성상 인플레이션 사이클(약 1Y), 자산 사이클(3~5Y), 인구
  구조(generational)가 자연스러운 단위. 4-tier(a)가 도메인 친화적.
- (c)는 cite check 비교 로직이 복잡 (range overlap). 운영 비용 vs 유연성.
- (d)는 비교 단순하지만 사람 읽기 비친화적.

권고: (a) 4-tier로 lock하고, 필요 시 ADR-0009 supersede로 확장.
