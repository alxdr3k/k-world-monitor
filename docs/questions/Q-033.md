# Q-033: 외부 플랫폼 auto cross-post timing (Substack / YouTube / X)

## Context

DEC-005에서 v0 외부 플랫폼 cross-post를 manual로 lock (운영자가 직접 게시,
cite footnote는 자체 사이트 URL anchor 사용). v1+에 plat별 auto cross-post
도입 시점 + workflow 결정 필요.

## Discussion

plat별 우선순위 / 비용:

| Plat | API 비용 | 우선순위 | Pre-req |
|---|---|---|---|
| Substack | 중 (API + email export) | v1.1 | newsletter format 활성 (Q-032) |
| YouTube | 中-高 (Data API + video upload + chapter) | v1.2 | youtube_long 활성 (Q-031 TTS + Q-032) |
| X (Twitter) | 中 (API v2 + tier 비용) | v1.3 | shorts 활성 또는 micro-format 신설 |
| LinkedIn | 중 | v2 | 추가 검토 — 1인 콘텐츠 audience 맞춤 |
| Threads / Mastodon | 낮 | v2 | 추가 검토 |

phasing 권고: **Substack auto → YouTube auto → X auto**

핵심 제약:
- 모든 cross-post 발행물의 cite footnote 가 자체 사이트 URL을 canonical
  anchor 로 가리켜야 함 (ADR-0022 INV-0022-2). plat별 API integration은
  body 안의 link transform 으로 보장
- 자체 사이트 ContentDraft `status` 가 `corrected` / `retracted` 로 전이 시
  cross-post 도 동일 state visible 갱신 필요 → plat API 의 edit / unpublish
  capability 의존
- API 비용 / rate limit 모니터링 — Anthropic API cost ledger (ADR-0006)
  와 별도 plat API ledger 필요

## Resolution

resolved 시 → DEC-### / ADR-####. v1 PUB-1B 트랙 진입 + Q-032 newsletter
phasing 진입 시점에 lock.
