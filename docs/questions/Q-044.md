# Q-044: NFR-008 audit log (raw text 0건 cloud 저장) 의 코드 enforcement 위치

## Context

본 Q 는 2026-05-13 comprehensive review 의 사용자 결정 필요 항목 #3 이다.

NFR-008 은 PRD §Non-functional 의 핵심 legal_safety 요구로, R2 upload
의 archive_policy 통과 audit log 와 raw third-party text 0건 cloud 저장을
의무화한다. 측정 방법으로 `policy_decisions` ledger 검사를 명시한다.

`migrations/sqlite/v1_schema.sql` 안에 `policy_decisions` 테이블은 정의
되어 있다 (decision_id, session_id, source_id, url, intended_action,
decision, gate_mode, risk_level, reason, intervention_id, created_at).
그러나 코드베이스 전수 grep 결과 **R2 upload 시점에 본 테이블에 INSERT
하는 구문이 존재하지 않는다** — `src/storage/r2/*` / `src/discovery/worker/
snapshot-fingerprint.ts` 어디에도.

NFR-008 의 measurement 가 작동하지 않는 상태이므로 P0-M1 게이트 검증
시점에 AC-032 (legal_safety) 의 evidence 가 비어 있다.

## Discussion

위 frontmatter `proposed_answer` 에 옵션 (a) ~ (c) 와 기본 권고 (b)+(c)
hybrid 가 적혀 있다. 추가 고려:

- ADR-0017 의 policy_gate mode 가 inline_block / inline_warn / batch_report
  중 어디로 R2 upload 결정 이 분류되는가? "Extract·Cache·Embed·Cloud
  upload=inline_block" 으로 PRD REQ-018 에 명시 — inline_block 으로 통일.
- policy_decisions row 의 `intended_action` enum 값이 정의되어 있는가?
  `src/utils/enums.ts` 검사 결과 IntendedAction enum 없음 — 본 Q resolution
  시 enum 추가 필요. 후보값: `r2_put` / `r2_get` / `extract` / `embed` /
  `cache` / `cloud_upload_generic`.
- AC-032 (NFR-008 audit) 의 TEST-032 위치가 `(planned)` 상태. 본 Q
  resolution 후 TEST-032 가 구현되어야 P0-M1 게이트 evidence 채울 수
  있음.

기본 권고 (b)+(c) hybrid 의 작업 순서:

1. `src/storage/audit/policy-decisions.ts` 신규 — `recordR2UploadDecision()`
   함수 (source_id, url, decision, gate_mode='inline_block', reason).
2. `src/utils/enums.ts` 에 IntendedAction enum 추가 (`r2_put`, `r2_get`,
   `extract`, `embed`, `cache`).
3. `src/discovery/worker/snapshot-fingerprint.ts` 의 r2Put 호출 직전/직후
   에 recordR2UploadDecision() 호출 — allowed / blocked 각 경로 모두.
4. `tests/unit/policy_decisions_audit_test.ts` 신규 — TEST-032 위치 lock.
5. `docs/current/DATA_MODEL.md` 의 policy_decisions 테이블 entry 갱신.
