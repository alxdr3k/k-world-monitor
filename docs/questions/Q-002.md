# Q-002: Dossier stale_after 기본값

## Context

ADR-0010 PRE-0010-2 — stale time 트리거의 임계값은 주제별로 다르다. 거시 지표는
월간 발표 주기(약 14d)가 자연스럽고, 감염병 동향은 주간 보고(7d), 지정학은 월간
이상(30d~).

ideation Round 3 Q2 보류 — Dossier 단위 default 분류가 필요한가, 또는
sub-category 별로 더 세분화해야 하는가.

## Discussion

- 카테고리별 default + dossier override는 운영 단순성 + 유연성 균형.
- "주제별 자동 추론"은 LLM 비용 + 잘못된 추론 시 cascade 위험.
- placeholder 30일은 ADR-0010 시점 임시값 (Q-002 미해결 동안 유효).

권고:
- topic categorization enum: macro_indicator | geopolitics | infectious_disease
  | technology | other
- 각 카테고리 default stale_after 14d / 30d / 7d / 30d / 30d
- dossier 단위 override는 frontmatter `stale_after_days: <int>` 로
