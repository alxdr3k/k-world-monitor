# Q-003: Publication 정정 ledger 트리거 정의

## Context

ADR-0003 INV-0003-7와 ADR-0008 INV-0008-3에 따라 Publication은 cite check
실패 시 corrected state로 전이해야 한다. 그러나 "어떤 변화가 cascade 트리거가
되는가"는 명시되지 않았다 (ideation Round 3 Q3 보류).

## Discussion

후보 트리거:

- (a) 인용된 claim의 claim_status가 confirmed → stale/disputed/retracted 전이
- (b) 인용된 scenario revision이 supersedes 당함 (새 revision 등장)
- (c) 운영자 명시 correction (오타 / 잘못된 표현 수정)
- (d) cite check rerun에서 horizon/unit/overclaim 신규 검출

(a)(b)(c) 셋 다 채택이 자연스럽다. (d)는 (a)와 겹침 — 별도 트리거 불필요.

또한 트리거별 cascade severity 구분 필요:

- alert only: (b) revision update (논리적으로 유효한 진화)
- corrected entry 강제: (a) stale, (c) 운영자 correction
- retracted cascade: (a) retracted
