# Q-029: ImpactAssessment v0 embedded dict vs v1 별도 Neo4j 노드

## Context

R25 GPT 메타 재검토에서 도출. ADR-0019 INV-0019-9에 단계화 명시됐지만 v0/v1
전이 시점 finalize 필요.

## Discussion

**v0 embedded dict** (현재 ADR-0019 결정):
- `scenario.impact_targets[]` + `scenario.impact_direction_by_target` (dict)
- pros: schema migration cost 미미, 정성 필드, v0 도입 부담 적음
- cons: target별 evidence/edge 추적 어려움 (모든 target이 한 scenario 노드의
  property)

**v1 별도 노드** (ADR-0019 INV-0019-9 + R25 GPT 권고):
- `:ImpactAssessment {target, direction, magnitude, confidence}` 노드
- edges: `:Scenario-[:HAS_IMPACT]->(:ImpactAssessment)-[:AFFECTS]->(:Target)`,
  `:ImpactAssessment-[:TRANSMITTED_THROUGH]->(:Channel)`
- pros: target별 evidence/edge 자연 추적 (e.g., specific claim → specific
  target의 direction), GDS centrality/community 적용 가능
- cons: schema migration cost (scenario revision 시 ImpactAssessment 노드도
  새 row + lineage edge), query 복잡도 ↑

권고: v0 embedded → v1 정식 노드. v1 전이 시점은 publication 누적 10건 + 사용자
의 "target별 evidence가 필요하다" 명시 시점. AGG-1A.2 슬라이스 진입 전에 lock.
