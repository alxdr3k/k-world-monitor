  | Layer | 메커니즘 | 위치 |

  | RSS feed 응답 캐시 | ETag / Last-Modified 조건부 fetch (304 시 body skip) | `src/discovery/scheduler/scheduler.ts:484-486` |

  부족한 부분:

  1. **개별 article URL (RSS item URL / manual intake URL) 의 conditional
     fetch 캐시**: 현재 crawl_state 는 RSS feed 단위로만 ETag / Last-Modified
     저장. RSS item 의 article URL 은 conditional header 미사용 — 재방문 시
     매번 full body 다운로드.
  2. **재방문 주기 결정 알고리즘**: 현재 정책 없음. 한 번 Snapshot 만들면
     다시 fetch 안 함 — **현 코드와 다름** (GPT 정정): done URL 도 RSS feed
     재 등장 시 다시 queue. 즉 암묵적 재방문 *열려 있고* 제어 mechanism *없음*.
  3. **변경 감지 알고리즘 — naïve hash diff 의 false-positive**: 광고
     trackback ID, timestamp, comment count, dynamic widget 등이 본문 외
     영역에서 매번 바뀌면 sha256 매번 다른 hash — semantic 변경 없는데도
     "변경됨" 처리.
  4. **HTTP Cache-Control / Expires header 활용 부재**: 서버 cache hint 무시.
  5. **discovery_queue.content_hash 의 dual-use design smell**: rss-worker
     INSERT 시 sha256(url), snapshot-fingerprint UPDATE 시 body sha256 으로
     overwrite. 같은 컬럼이 pending state 에서 URL hash, done state 에서
     body hash. 재방문 정책 도입 시 혼란.

  event-driven trigger:
  - `manual_refresh` (운영자 명시 `pipeline refresh <snap_id>`)

    이전 라운드 visited URL 강제 재검증)

  #### (2) 변경 감지 알고리즘

  채택: **Mozilla Readability + normalize + sha256 (canonical_text_hash)** +
  **raw_body_hash 별도 보존**.

  - **raw_body_hash** = 원본 bytes sha256 (현재 content_hash 그대로)
  - **canonical_text_hash** = HTML 본문만 추출 (Readability) → normalize
    (whitespace collapse / lowercase / punctuation strip) → sha256
  - **변경 판단 1차 기준**: canonical_text_hash diff (의미 변경)
  - **변경 판단 2차 기준**: raw_body_hash diff (광고/timestamp 변경 — v1+
    `source_changed_minor` 분류)
  - PDF / API / CSV: canonical_text_hash 미사용 — raw hash 또는 dataset
    vintage checksum
  - **SimHash / MinHash 는 P1+ phasing** — MVP 단계 canonical exact hash
    만, 유사도 기반 변경 감지는 관측 데이터 누적 후

  #### (3) HTTP cache header 활용

  채택: **ETag / Last-Modified 만** (3b). Cache-Control / Expires 는 P1+
  (서버 hint 신뢰도 검증 후).

  #### (4) Content extraction 도구 선택

  채택: **Mozilla Readability (Apache-2.0, JS 네이티브)**. 본 repo bun
  환경 호환 + license 호환. Trafilatura (Apache-2.0, Python — 본 메타
  리뷰에서 GPL-3.0 표기 정정) 는 P0-M3 Data Science Module 도입 후 P1+
  검토.

  **DOM parser 선택 (Readability 의존)**: linkedom (faster, lighter) vs
  jsdom (mature, heavier). P0 권고 = linkedom.

  #### (5) 구현 슬라이스 우선순위

  - **INFRA-1B.10** (가장 먼저, GPT 권고 scope 확대): article-level

  - **INFRA-1B.9**: canonical text hash (Readability 도입)

  ### discovery_queue.content_hash dual-use cleanup (사용자 결정 필요 항목 #3 의 일부)

  현재 schema design smell. 옵션:

  - **(A)** 컬럼 제거 — rss-worker INSERT 시 미작성, snapshot-fingerprint
    UPDATE 시도 미작성. body_hash 는 Snapshot.content_hash 에서 만 참조.
  - **(B)** 컬럼 분리 — `discovery_queue.url_hash` (pending state, sha256(url))
    + `discovery_queue.body_hash` (done state, body sha256). 의미 명확.
  - **(C)** 현 상태 유지 + 주석 보강 — design smell 인정.

  기본 권고: **(A) 제거**. body_hash 는 Snapshot.content_hash 에서, url_hash
  는 url 컬럼 자체로 충분 (sha256(url) 은 추가 정보 없음).

  ### Snapshot Neo4j schema 영향 (사용자 결정 필요 항목 #3)

  raw_body_hash + canonical_text_hash 분리 권고 시 Neo4j Snapshot node 의
  property 추가:

  - **옵션 A**: raw_body_hash 만 UNIQUE (현재 content_hash 그대로) +
    canonical_text_hash 는 normal property
  - **옵션 B**: 둘 다 UNIQUE (composite or separate constraint)

  기본 권고: **(A)** — byte-level identity 보존 (Snapshot 자체는 raw bytes
  의 fingerprint 라는 정의 유지). canonical_text_hash 는 변경 감지 / dedup
  index 로 활용.

  ### snapshot_diff semantics reflow (사용자 결정 필요 항목 #4 — ADR-0010 본문 갱신)

  ADR-0010 (stale 트리거 3 종 — time / snapshot_diff / counterclaim) 의
  "snapshot_diff" 정의가 raw vs canonical 명시 X. reflow 필요:

  - **옵션 X**: snapshot_diff = canonical_text_hash diff (의미 변경 trigger)
    + raw_body_hash diff 는 별도 `source_changed_minor` alert (v1+)
  - **옵션 Y**: snapshot_diff = raw_body_hash diff (현재 implicit) — canonical
    hash 는 별도 dedup 만
  - **옵션 Z**: 둘 다 trigger — raw diff 면 minor, canonical diff 면 major

  기본 권고: **(X)** — 의미 변경이 stale trigger 의 의도와 일치. raw 변경
  (광고 / timestamp) 은 false-positive trigger.

  Reflow 위치:
  - (i) ADR-0010 본문 직접 갱신 — single source of truth
  - (ii) 별도 ADR (가칭 — Snapshot Diff Semantics) — ADR-0010 supersede
  - (iii) DEC 로 lightweight lock — ADR 본문 변경 없음

  기본 권고: **(i) ADR-0010 본문 갱신** — supersede 까지는 과도, lightweight
  DEC 도 의미적 변경이 ADR scope 안이라 ADR 본문이 적합.
