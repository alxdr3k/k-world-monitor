### ScenarioRevision vs round_synthesis boundary (사용자 결정 항목)

- round_synthesis = 라운드 단위 working memory (round 종료 시 archive,
  영구 보존 audit log)
- ScenarioRevision = 시나리오 evolution 의 영구 lineage (operator 가
  round_synthesis 로부터 promote)
- transition rule: round close 시점에 operator 명시 promote → round_
  synthesis.promoted_to_scenario_revision_id FK + ScenarioRevision.source
  _round_id FK 양방향

### parent_round_id branching 의 round_no semantics (사용자 결정 항목)

옵션:
- (a) parent 별 round_no reset (2A = round 2, 3A = round 3, 2B = round 2, 3B = round 3)
- (b) session 단조 증가 (1 / 2A=2 / 2B=3 / 3A=4 / 3B=5)
- (c) depth + branch_id 분리 (depth=2/branch_id=A, depth=2/branch_id=B)

기본 권고: (c). schema 에 이미 depth + branch_id 두 컬럼 분리됨.

### 운영자 vs AI 책임 분리

- ScenarioRevision lock = operator 명시 (ADR-0025 INV-0025-4)
- EditorialIntent lock = operator 명시
- Scenario branches 자동 ranking = deterministic algorithm 만 (LLM 우선
  순위 결정 금지)
- round_synthesis 작성 = AI 가능 + operator 검토 의무
