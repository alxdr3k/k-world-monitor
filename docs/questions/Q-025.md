# Q-025: 외부 repo 부트스트랩 cadence

## Context

R24 자체 audit에서 신설. 1인 운영 환경에서 INFRA-1A.1 완료 후 어느 cadence로
진행할지 가이드 필요. 시간 박싱 없이 진행한다는 capacity policy는 유지하되,
사용자가 더 빠르게/느리게 갈지 결정 기준 마련.

## Discussion

권고 cadence (week 1-9, 운영자 우선순위 반영):
- **Week 1-2**: ADR scaffold(완료, 본 reset) + Neo4j Docker init + SQLite
  migration v1 + Tier A seed 10-15개 (Q-021 부분 reduction)
- **Week 3-4**: collection_gate (ADR-0017 source_policy 3 필드) + fetch worker
  + first claim extraction (Haiku 4.5 1차)
- **Week 5-6**: first dossier (예: `productivity_upside` 또는 `energy_abundance`)
- **Week 7-8**: first scenario(impact_targets + transmission_channels) +
  first thesis (stance + market_stance, bidirectional framing 강제)
- **Week 9+**: first publication (cite check 5+1 + EvidencePack 4-section)

운영자 우선순위에 따라 단축(예: ADR-0019 일부 v1 deferred) 또는 연장(예: Tier
A seed 30~50개 모두 v0 입력) 가능.
