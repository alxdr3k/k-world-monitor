# Q-020: Neo4j Community GPL v3 boundary

## Context

ADR-0014에서 Neo4j Community Edition(GPL v3, single-user) 채택과 의도적 lock-in
수용을 결정했다. R21 GPT 종합 리뷰에서 도출된 신규 Open Question — GPL v3
contagion이 미래 시점에 문제가 될 수 있는 경우 명문화 필요.

미해당 시나리오 (1인 internal use):
- 콘텐츠 production 내부 도구로만 사용
- DB 자체를 사용자에게 배포하지 않음
- 발행 콘텐츠(블로그/유튜브)는 DB와 별개 산출물

해당 가능 시나리오 (재검토 필요):
- (a) DB를 제품에 embed해 배포 (e.g., desktop app for collaborators)
- (b) Neo4j 위 build한 Cypher procedure를 공개 fork로 분리 (서드파티에 배포)
- (c) 콘텐츠 발행 시 DB가 "distributable component"로 간주되는 케이스 (예:
  "open data dump"로 SQL dump + Neo4j dump 동봉 배포)

## Discussion

- 1인 internal use는 GPL v3 contagion 없음 — GPL은 redistribution 시점에
  trigger.
- 매 major upgrade 시 license file 확인 의무 (R17/Q12-2 결정)
- 향후 (a)(b)(c) 가능성 발생 시 별도 ADR로 boundary 갱신 + 라이선스 검토 후
  진행

권고: 1인 internal use 범위로 명시하고, (a)(b)(c) trigger 발생 시 별도 ADR로
처리.
