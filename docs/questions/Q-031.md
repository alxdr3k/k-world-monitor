# Q-031: TTS v1 timing + provider — 외주 vs 자체

## Context

DEC-005에서 v0 turn-key publish scope 결정 시 TTS를 deferred로 분류. v0
활성 format은 blog_long 1개이므로 음성 콘텐츠 자체가 v0 부재. v1+에 
youtube_long / shorts auto-generate 활성 시 (Q-032 phasing) TTS pipeline
도입 시점과 provider 결정 필요.

## Discussion

후보:

1. **외주 API (ElevenLabs / OpenAI TTS / Google Cloud TTS)**:
   - pros: setup 부담 ↓, quality 고품질, scale auto, voice clone 옵션
   - cons: cost ~$0.10~0.30/min (1편 10분 → $1~3, 주 3편 → 월 $40~120),
     vendor lock-in 추가, 정책 변경 / 음성 cloning 윤리 이슈

2. **자체 (Coqui / XTTS-v2 / Bark self-host)**:
   - pros: vendor lock-in 회피, 무제한 generation, voice clone 자유도
   - cons: GPU 필요 (self-host 인프라 추가), quality variance, 모델 license
     주의, 운영 부담 ↑ (1인 + LLM agent 환경엔 부담)

3. **사람 voice acting 외주**:
   - pros: quality 최고, 시청자 신뢰 ↑
   - cons: scale 어려움, cost 高 (시간당 $50~200), 자동화 불가

v1 default 권고: **(1) 외주 API + R2 호스팅**
- ElevenLabs (voice quality 고품질) 또는 OpenAI TTS (cost 낮음, $0.015/1K
  char ≈ $0.10~0.20/10분) — v1 spike로 quality 비교
- 산출물(mp3/aac)은 R2 permitted artifact (ADR-0012) — self 산출물 카테고리
- Astro Content Collection schema 확장: youtube_long / shorts frontmatter에
  `audio_url`, `duration_sec` 추가
- RSS audio feed (`<enclosure>` 태그) — 팟캐스트 / YouTube 영상 embed 모두
  지원

미해결:
- 한국어 + 영어 dual track 지원 여부 (v1+ 자동 번역과 함께)
- voice persona (브랜드 일관성) — voice clone vs preset
- shorts 60초 자체는 TTS 없이 자막만 + BGM 도 옵션
- youtube_long 10분 영상의 chart / b-roll / visual pacing 은 별도 토픽
