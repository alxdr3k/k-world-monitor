# Q-047: Secret management — Doppler vs `.env` (OpenAI / Anthropic / Google / R2 API key 4종)

## Context

본 Q 는 2026-05-13 comprehensive review 의 사용자 결정 필요 항목 #6 이다.

PUB-1A.5 turn-key MVP 발행 시점에 4 종 secret 이 동시에 필요:
- OpenAI API key (Tier 2 default + Tier 1 escalate + Tier 0 GPT-5.5 Pro xthink)
- Anthropic API key (cross-vendor review + Sonnet 4.6 한국어 override + Opus
  4.7 xhigh)
- Google AI Studio key (Gemini 2.5 Flash 탐색 보조 — 선택)
- Cloudflare R2 access key + secret key (permitted artifact upload)
- (간접) Cloudflare Pages git integration token — git push 시점 자동

`docs/05_RUNBOOK.md` 의 Publishing Site Deployment 섹션이 "Doppler 또는
환경 변수 등록 의무" 로 ambiguous 표현. 본 Q 가 그 ambiguity 를 closure.

## Discussion

위 frontmatter `proposed_answer` 의 옵션 (a) ~ (d) 와 기본 권고 (a) +
direnv 가 적혀 있다. 추가 고려:

- GitHub Actions cron 환경: secret 은 repository secret 으로 등록 — 본
  Q 와 직교 (GH secret 은 항상 사용). 본 Q 는 로컬 dev / cron host 시점.
- `direnv` 도입 시 `.envrc` 파일 + `direnv allow` 필요. local dev 시점
  자동화 가능하지만 1 인 운영 환경에서 overhead 약함.
- Cloudflare R2 credential 은 wrangler CLI 가 별도 management — `.env`
  와 wrangler config 가 분리될 수 있음. 본 Q resolution 시 wrangler
  config 도 같이 lock.
- PR / CI 검증 단계에서 `.env` 가 실수로 commit 되지 않도록 `.gitignore`
  + `git secrets` 또는 pre-commit hook 검토.

기본 권고 (a) 의 작업 순서:

1. `.env.example` 신규 — 4 종 key 의 변수명 + placeholder.
2. `.gitignore` 에 `.env` 확인 (이미 있는지 검사).
3. `docs/05_RUNBOOK.md` 의 "Doppler 또는 환경 변수" 표현을 ".env file +
   direnv (local) / GitHub Actions secret (cron)" 으로 lock.
4. pre-commit hook 또는 `git secrets` 도입 검토 (선택).

## 사용자 결정 필요 항목

1. Secret management 방법 — (a) `.env` + direnv / (b) Doppler / (c)
   1Password CLI / (d) GPG-encrypted file 중 선택
2. GitHub Actions cron 환경의 secret 주입 — repository secret 직접 사용
   동의 여부
3. R2 credential 의 wrangler config 와 `.env` 의 분리 정책 — 둘 다 사용
   vs 단일화 결정
4. pre-commit hook 또는 `git secrets` 도입 여부 (선택)
