# Q-042: NFR-003 5-hop trace 의무를 ADR-0025 10-stage 도입 이후 어떻게 reflow 할 것인가

## Context

본 Q 는 2026-05-13 comprehensive review (docs/10_PROJECT_RETROSPECTIVE.md
"Comprehensive Review — 2026-05-13" 섹션) 의 사용자 결정 필요 항목 #1 이다.

PRD `01_PRD.md:156` NFR-003:

> traceability — 콘텐츠 한 문장 → 원 source 까지 5단계 이내 (Publication →
> ContentDraft → Thesis → Scenario → Claim → Snapshot → Source) — 9-stage
> 안에서 5단계 이내 유지(선택적 단계 skip)

ADR-0025 가 Scenario 와 Thesis 사이에 EditorialIntent 를 anchor 로 추가
하면서 정식 trace path 가 한 hop 늘어났다. PRD NFR-003 의 "5단계 이내"
표현은 ADR-0011 9-stage 시점에 정의됐고, ADR-0025 supersede 이후 본문
갱신이 누락되어 있다.

이 NFR 은 publication preflight 5-check (ADR-0015) 의 일부로 cite check
파이프라인에서 측정된다 (REQ-013, AC-018, AC-028). 본 Q 가 resolved 되기
전까지는 NFR-003 의 정확한 hop 수와 path 구성이 ambiguous 하므로
PUB-1A.2 (Cite Check 5+1) 구현 직전에 lock 이 필요하다.

## Discussion

위 frontmatter `proposed_answer` 에 옵션 (a)~(d) 와 기본 권고 (a) 가
적혀 있다. 추가 고려:

- 본 NFR 은 manual_claim_entry path (ADR-0018) 와도 상호작용한다. manual
  claim 은 Snapshot 이 없으므로 trace 가 한 hop 짧다. (c) 옵션 일반화 시
  manual claim 의 예외 처리도 같은 메커니즘 안에 들어간다.
- AC-018 (`docs/06_ACCEPTANCE_TESTS.md`) 의 "5+1 cite check" 본문은 5-hop
  / 7-hop 어느 쪽이든 implementation 측면에서는 graph traversal depth 카운
  트만 바뀐다 — 큰 비용 차이 없음.
- ADR-0027 (claim-level evidence_role classification) 이 도입되면 Dossier
  안에서 supporting/opposing/monitoring evidence role 별 trace 의무가 추가
  될 수 있음. 본 Q 와 직접 충돌하지는 않지만 후속 NFR 추가 시 다시 hop
  계산이 필요.
