# Q-039: pollEligibleSources Phase 1/2 streaming

## Context

PR #25 adversarial review (iter 1 finding F-13)에서 식별. v0
operational scale에서는 영향이 없지만 source pool 확장 시 메모리
ceiling이 명확해진다.

## Discussion

### 현재 구현 한계

```typescript
// scheduler.ts:111-170
const fetchResults = await Promise.allSettled(
  targets.map((src) => runWithPool(hostname, async () => {
    const res = await safeFetch(src.feed_url, fetchOpts);
    return { ..., body: res.body };  // 모든 body가 simultaneous resident
  }))
);
// Phase 2 (write) runs only after ALL fetches settle
```

worst-case = `targets.length × MAX_BYTES.rss` 메모리 동시 보관.

### 해법 후보

- (a) chunked allSettled (예: 10개씩 처리, write 후 다음 batch fetch)
- (b) async iterator pattern (`for await`) — fetch 완료 즉시 write
- (c) p-queue 기반 work-stealing

(a)가 v0와 가장 호환되고 implement 비용 최소.

### 사용자 결정 필요 항목

1. Q를 dropped로 resolve할지 (v0 scale에서는 문제 안 됨)
2. INFRA-1B.2.x로 promote할지
