  #### (1) CLI 라우팅 6 경로

  모든 사용자 질문은 6 경로 중 하나로 라우팅:

  | 경로 | flag | 동작 |
  |---|---|---|
  | new_session | `--new-session` 또는 `pipeline research start` | research_session 신규 + 자동 Round 1 생성 + active context set |
  | continue_round | (flag 없음 + active context 존재) | active session 의 active round 에 ResearchTurn append |
  | new_round | `--new-round --mode <mode> --goal <text>` | 같은 session 의 새 ExplorationRound + Turn |
  | branch_round | `--branch-from <round_id>` | 같은 session 의 새 ExplorationRound (parent_round_id = 명시), branch_id 자동 (A/B/...) |
  | fork_session | `--fork-session --from-session <session_id>` | 신규 research_session (forked_from_session_id = 명시) + Round 1 |
  | answer_only | `--answer-only` 또는 routing miss | turn 만 기록, search_run / synthesis 미생성 |
  | return | `pipeline session use <id>` + `pipeline round use <id>` | **active context 변경 only — ResearchTurn 미생성** (context-switch operation, route enum 외). research_turn.route CHECK constraint 에 미포함. 다음 ask 부터 변경된 context 적용 |

  #### (3) ResearchTurn schema 신규 (migration v8 안 통합)

  ```sql
  CREATE TABLE research_turn (
    turn_id              TEXT PRIMARY KEY,  -- turn_<ULID>
    session_id           TEXT NOT NULL REFERENCES research_session(session_id),
    round_id             TEXT REFERENCES exploration_round(round_id),
    parent_turn_id       TEXT REFERENCES research_turn(turn_id),
    user_message         TEXT NOT NULL,
    interpreted_intent   TEXT NOT NULL,
    route                TEXT NOT NULL
      CHECK (route IN (
        'new_session', 'continue_round', 'new_round',
        'branch_round', 'fork_session', 'answer_only'
      )),
    route_source         TEXT NOT NULL
      CHECK (route_source IN ('flag', 'active_context', 'classifier_default')),
    classifier_suggested_route   TEXT,  -- AI intent classifier output
    classifier_confidence        REAL,
    classifier_reason            TEXT,
    answer_summary       TEXT,
    response_artifact_id TEXT,  -- snap_<ULID> / dos_<ULID> / scn_<ULID> / etc
    cost_usd             REAL,
    created_at           TEXT NOT NULL,  -- millis-bearing ISO (DEC-019)
    completed_at         TEXT
  );
  CREATE INDEX research_turn_session_idx ON research_turn(session_id);
  CREATE INDEX research_turn_round_idx ON research_turn(round_id);
  CREATE INDEX research_turn_created_at_idx ON research_turn(created_at);
  ```

  - `parent_turn_id` — 같은 round 안의 conversation thread (Turn 1 의 follow-
    up Turn 2)
  - `route` — 6 enum (위 표)
  - `route_source` — 명시 flag / active context / classifier default 추적
  - `classifier_*` 3 컬럼 — AI intent classifier 가 제시한 route + 사용자가
    채택했는지 audit log

## Discussion

위 frontmatter `proposed_answer` 에 6 라우팅 경로 + flag priority + schema
신규 + CLI 명령 표준 + AI intent classifier 정리.

핵심 원칙:
- **DB 에는 항상 명시적** — 모든 질문은 session_id / round_id / turn_id 로 기록
- **CLI UX 는 active context 제공** — operator 매번 ID 입력 회피
- **명시 flag > active context > classifier** — 우선순위 lock
- **classifier 는 보조** — auto routing 사고 방지

### Boundary with Q-050 (research orchestration layer)

Q-050 = research orchestration layer (ExplorationRound + RoundSynthesis +
SearchOrchestrator + RoundContextPack). Q-051 = 그 layer 위의 CLI UX +
ResearchTurn (Turn = round 안의 개별 user question/answer).

ResearchTurn 이 ExplorationRound 의 child 객체. 한 round 안에 N turns:

```
ResearchSession
  └─ ExplorationRound (Q-050)
       └─ ResearchTurn (Q-051)
            └─ RoundSearchRun (Q-050)
                 └─ RoundCandidateUrl (Q-050)
```

즉 turn 은 user-facing 단위, round 는 orchestration 단위, session 은
조사 주제 단위.

### 운영자 명시 vs AI classifier 분리

- **명시 routing** = operator 의 "이 질문은 어디로" 명시 의도. 우선순위 최고.
- **active context** = 직전 CLI session 의 state. 편의용.
- **classifier** = LLM 의 "이 질문은 topic shift" 추측. 보조용. 절대 단독
  override X. sensitive route (fork / branch) 는 사용자 확인 의무.
