# Q-008: Thesis ID 체계

## Context

ADR-0011에서 Thesis 객체를 9-stage에 추가했다. Scenario revision을 압축한
재사용 가능한 핵심 주장으로, 같은 Thesis가 4 format ContentDraft (blog_long /
youtube_long / shorts / newsletter)에 재사용된다.

같은 Thesis의 dedupe와 lineage 추적을 위해 ID 체계가 lock되어야 한다.

후보:
- (a) `ths_<sha256[0:10]>` — body content hash 기반. 같은 본문은 같은 ID
- (b) `ths_<ULID>` — 단조 증가 식별자. 본문 변경에도 lineage 유지 (별도 supersedes
  edge로 lineage)
- (c) Thesis 객체 없이 ContentDraft 내부 `thesis_text` 필드로 — 객체 제거

## Discussion

- (c)는 Round 12/Q7 GPT 메타리뷰에서 채택된 Thesis 객체화의 의미를 부정.
  같은 Thesis 4번 재작성 비용 발생. 폐기.
- (a)는 hash 기반이라 dedupe 자연. 단 본문 1자 수정이 새 ID로 폭증 위험.
- (b)는 supersedes edge(ADR-0013) lineage 추적과 일관. ULID 권고.

권고: (b) `ths_<ULID>` + 본문 hash는 별도 `body_sha256` 필드. dedupe는 hash로,
lineage는 ID + edge로.
